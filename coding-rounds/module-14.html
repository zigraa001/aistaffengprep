1. Trie Implementation

Core Trie — Insert, Search, StartsWith, Delete ▼

A trie (prefix tree) stores strings character-by-character. Each edge represents a character, and each node may mark the end of a word. Tries give O(L) lookup where L is the word length, independent of how many words are stored — unlike hash tables which degrade under collisions.

Why tries matter in interviews: They are the canonical solution for prefix-based queries (autocomplete, spell check, IP routing). Google's search suggestion box is conceptually a trie with ranking.

Full Implementation (Python)

class TrieNode:
    __slots__ = ('children', 'is_end')
    def __init__(self):
        self.children = {}   # char -> TrieNode
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self._find(word)
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        return self._find(prefix) is not None

    def _find(self, prefix: str):
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node

    def delete(self, word: str) -> bool:
        """Returns True if word was deleted. Uses post-order cleanup."""
        def _delete(node, word, depth):
            if depth == len(word):
                if not node.is_end:
                    return False
                node.is_end = False
                return len(node.children) == 0  # can prune?
            ch = word[depth]
            if ch not in node.children:
                return False
            should_prune = _delete(node.children[ch], word, depth + 1)
            if should_prune:
                del node.children[ch]
                return not node.is_end and len(node.children) == 0
            return False
        _delete(self.root, word, 0)
        return True

2. Classic Trie Problems

LeetCode #211 — Design Add and Search Words (Wildcard) ▼

Problem: Design a data structure that supports addWord(word) and search(word) where search can contain . as a wildcard matching any single character.

Key insight: Standard trie insert; for search, when you hit a ., branch into all children recursively.

class WordDictionary:
    def __init__(self):
        self.root = TrieNode()

    def addWord(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        def dfs(node, i):
            if i == len(word):
                return node.is_end
            if word[i] == '.':
                return any(dfs(child, i + 1)
                           for child in node.children.values())
            if word[i] not in node.children:
                return False
            return dfs(node.children[word[i]], i + 1)
        return dfs(self.root, 0)

Time: O(L) for addWord. O(26^L) worst case for search with all dots, but in practice much faster due to pruning.

LeetCode #212 — Word Search II (Trie + Backtracking) ▼

Problem: Given an m×n board of characters and a list of words, find all words that can be formed by sequentially adjacent cells (no cell reused per word).

Why trie: A naive approach searches for each word independently (O(words × m×n × 4^L)). Building a trie of all target words lets you search for all words simultaneously in a single board traversal.

def findWords(board, words):
    root = TrieNode()
    for w in words:                # build trie of target words
        node = root
        for ch in w:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.word = w              # store full word at terminal

    result = []
    rows, cols = len(board), len(board[0])

    def dfs(r, c, node):
        ch = board[r][c]
        if ch not in node.children:
            return
        child = node.children[ch]
        if hasattr(child, 'word') and child.word:
            result.append(child.word)
            child.word = None      # avoid duplicates

        board[r][c] = '#'          # mark visited
        for dr, dc in ((0,1),(0,-1),(1,0),(-1,0)):
            nr, nc = r + dr, c + dc
            if 0 <= nr < rows and 0 <= nc < cols and board[nr][nc] != '#':
                dfs(nr, nc, child)
        board[r][c] = ch           # restore

        if not child.children:     # prune empty branches
            del node.children[ch]

    for r in range(rows):
        for c in range(cols):
            dfs(r, c, root)
    return result

Optimization: The del node.children[ch] prune step is critical. Without it, you repeatedly explore dead branches. This brings the solution from TLE to accepted.

LeetCode #1268 — Search Suggestions System (Autocomplete) ▼

Problem: Given an array of products and a search word, after each character typed return the top 3 lexicographically smallest matches.

Approach: Sort products, build a trie, store sorted words at each node (or collect via DFS). Alternatively, use binary search on the sorted list for each prefix — simpler and often preferred in interviews.

def suggestedProducts(products, searchWord):
    products.sort()
    result, prefix = [], ""
    for ch in searchWord:
        prefix += ch
        # Binary search for leftmost match
        import bisect
        i = bisect.bisect_left(products, prefix)
        suggestions = []
        for j in range(i, min(i + 3, len(products))):
            if products[j].startswith(prefix):
                suggestions.append(products[j])
        result.append(suggestions)
    return result

Interview tip: Mention both approaches. The trie approach is O(total chars) to build but O(1) per prefix lookup. The binary search approach is O(n log n) sort + O(L log n) per query. For a one-shot query, binary search wins; for a persistent autocomplete service, the trie wins.

3. Segment Tree

Build, Query, Update for Range Sum ▼

When to use: You need both range queries (sum/min/max over [l, r]) and point or range updates. A prefix sum handles range queries in O(1) but updates cost O(n). A segment tree gives O(log n) for both.

Real-world usage: Database engines use segment-tree-like structures for range index scans. Competitive programming staple for problems involving mutable ranges.

graph TD N1["[0,3] sum=10"] --> N2["[0,1] sum=3"] N1 --> N3["[2,3] sum=7"] N2 --> N4["[0] val=1"] N2 --> N5["[1] val=2"] N3 --> N6["[2] val=3"] N3 --> N7["[3] val=4"]

Segment tree for array [1, 2, 3, 4]. Each node stores the sum of its range.

Compact Array-Based Implementation

class SegmentTree:
    """Range sum queries + point updates in O(log n)."""
    def __init__(self, nums):
        self.n = len(nums)
        self.tree = [0] * (2 * self.n)
        # Build: fill leaves, then internal nodes
        for i in range(self.n):
            self.tree[self.n + i] = nums[i]
        for i in range(self.n - 1, 0, -1):
            self.tree[i] = self.tree[2*i] + self.tree[2*i+1]

    def update(self, i, val):
        """Set nums[i] = val."""
        i += self.n
        self.tree[i] = val
        while i > 1:
            i //= 2
            self.tree[i] = self.tree[2*i] + self.tree[2*i+1]

    def query(self, l, r):
        """Return sum of nums[l..r] inclusive."""
        res = 0
        l += self.n
        r += self.n + 1
        while l < r:
            if l & 1:
                res += self.tree[l]
                l += 1
            if r & 1:
                r -= 1
                res += self.tree[r]
            l //= 2
            r //= 2
        return res

Complexity: Build O(n), query O(log n), update O(log n). Space O(2n) = O(n).

Why this implementation? The iterative bottom-up approach (often called "Codeforces-style") avoids recursion overhead and is half the code of the recursive version. Node i's children are 2i and 2i+1. Leaves are at indices [n, 2n).

4. Fenwick Tree (Binary Indexed Tree)

Point Update + Prefix Query ▼

A Fenwick tree supports prefix sum queries and point updates both in O(log n), using only a flat array of size n+1. It is simpler and has lower constant factors than a segment tree, but less general (no lazy propagation, harder to adapt to min/max).

Implementation

class FenwickTree:
    """1-indexed Binary Indexed Tree for prefix sums."""
    def __init__(self, n):
        self.n = n
        self.tree = [0] * (n + 1)

    def update(self, i, delta):
        """Add delta to index i (1-indexed)."""
        while i <= self.n:
            self.tree[i] += delta
            i += i & (-i)    # add lowest set bit

    def query(self, i):
        """Return prefix sum [1..i]."""
        s = 0
        while i > 0:
            s += self.tree[i]
            i -= i & (-i)    # remove lowest set bit
        return s

    def range_query(self, l, r):
        """Return sum [l..r] (1-indexed)."""
        return self.query(r) - self.query(l - 1)

When to choose: If you only need prefix sums + point updates, prefer Fenwick (simpler, faster). If you need range min/max, lazy propagation, or arbitrary range operations, use a segment tree.

5. Advanced Data Structures Overview

Skip Lists, Bloom Filters, Count-Min Sketch ▼

Skip List

A randomized structure layering sorted linked lists with express lanes. Expected O(log n) search/insert/delete. Used by Redis for sorted sets — simpler than balanced BSTs, supports range scans, and amenable to lock-free concurrency.

Bloom Filter

A space-efficient probabilistic set: no false negatives, possible false positives. Uses k hash functions mapping to a bit array of size m. False positive rate ~(1 - e^(-kn/m))^k.

Uses: Chrome safe browsing, Cassandra (skip disk reads for missing keys), Medium (avoid re-recommending articles).

Count-Min Sketch

A probabilistic frequency table using d hash functions × w counters. Query returns minimum across d positions (overcounts possible, undercounts not). Used for network anomaly detection and trending topic identification.
