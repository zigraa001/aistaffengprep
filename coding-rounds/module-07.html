Top-K Problems

Kth Largest Element ▼ 
import heapq

def find_kth_largest(nums, k):
    """
    Find kth largest using min heap of size k.
    Time: O(n log k), Space: O(k)
    """
    # Keep k largest elements
    min_heap = []

    for num in nums:
        heapq.heappush(min_heap, num)
        if len(min_heap) > k:
            heapq.heappop(min_heap)  # Remove smallest

    return min_heap[0]  # Kth largest is the smallest in heap

# Alternative: O(n) average with quickselect
# Caution: O(n^2) worst case without randomization. Mutates input.
import random

def find_kth_largest_quickselect(nums, k):
    k = len(nums) - k  # Convert to kth smallest index

    def quickselect(l, r):
        # Randomize pivot to avoid O(n^2) on sorted input
        rand_idx = random.randint(l, r)
        nums[rand_idx], nums[r] = nums[r], nums[rand_idx]

        pivot, p = nums[r], l
        for i in range(l, r):
            if nums[i] <= pivot:
                nums[p], nums[i] = nums[i], nums[p]
                p += 1
        nums[p], nums[r] = nums[r], nums[p]

        if p > k: return quickselect(l, p - 1)
        elif p < k: return quickselect(p + 1, r)
        else: return nums[p]

    return quickselect(0, len(nums) - 1)

import heapq

def merge_k_lists(lists):
    """
    Merge k sorted linked lists.
    Time: O(n log k), Space: O(k)
    """
    # Min heap of (value, index, node)
    heap = []

    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst.val, i, lst))

    dummy = ListNode()
    current = dummy

    while heap:
        val, i, node = heapq.heappop(heap)
        current.next = node
        current = current.next

        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next

Two Heaps Pattern

Why Two Heaps?

The Two Heaps pattern is one of the most important Grokking patterns. It lets you efficiently track the median of a dynamic dataset by splitting it into a "smaller half" (max-heap) and a "larger half" (min-heap). The median is always accessible in O(1) from the tops of the heaps. This pattern appears in streaming data, sliding windows, and optimization problems.

Find Median from Data Stream (LeetCode #295) ▼

Problem: Design a data structure that supports adding integers and finding the median of all elements added so far.

Key Insight: Maintain two heaps. The max-heap stores the smaller half, the min-heap stores the larger half. The median is always at the boundary.

    subgraph MinHeap["Min-Heap (larger half)"]

Invariants

  • Every element in max-heap ≤ every element in min-heap
  • Size difference between heaps is at most 1
  • If odd total, the max-heap has one extra element (median = max-heap top)
  • If even total, median = average of both tops
import heapq

class MedianFinder:
    """
    LeetCode #295 - Find Median from Data Stream

    Two heaps:
      - max_heap (negated): stores smaller half
      - min_heap: stores larger half

    addNum: O(log n)
    findMedian: O(1)
    """
    def __init__(self):
        self.small = []  # max-heap (negate values)
        self.large = []  # min-heap

    def addNum(self, num: int) -> None:
        # Step 1: Push to max-heap (smaller half)
        heapq.heappush(self.small, -num)

        # Step 2: Ensure max of small <= min of large
        if self.small and self.large and (-self.small[0]) > self.large[0]:
            val = -heapq.heappop(self.small)
            heapq.heappush(self.large, val)

        # Step 3: Balance sizes (small can have at most 1 more)
        if len(self.small) > len(self.large) + 1:
            val = -heapq.heappop(self.small)
            heapq.heappush(self.large, val)
        elif len(self.large) > len(self.small):
            val = heapq.heappop(self.large)
            heapq.heappush(self.small, -val)

    def findMedian(self) -> float:
        if len(self.small) > len(self.large):
            return -self.small[0]
        return (-self.small[0] + self.large[0]) / 2.0


# Usage:
# mf = MedianFinder()
# mf.addNum(1)    # [1]          median = 1
# mf.addNum(2)    # [1, 2]       median = 1.5
# mf.addNum(3)    # [1, 2, 3]    median = 2
// Java implementation using PriorityQueue
import java.util.*;

class MedianFinder {
    // Max-heap for smaller half (default PriorityQueue is min-heap, use reverseOrder)
    PriorityQueue<Integer> small = new PriorityQueue<>(Collections.reverseOrder());
    // Min-heap for larger half
    PriorityQueue<Integer> large = new PriorityQueue<>();

    public void addNum(int num) {
        small.offer(num);

        // Ensure ordering: max of small <= min of large
        if (!small.isEmpty() && !large.isEmpty() && small.peek() > large.peek()) {
            large.offer(small.poll());
        }

        // Balance sizes
        if (small.size() > large.size() + 1) {
            large.offer(small.poll());
        } else if (large.size() > small.size()) {
            small.offer(large.poll());
        }
    }

    public double findMedian() {
        if (small.size() > large.size()) {
            return small.peek();
        }
        return (small.peek() + large.peek()) / 2.0;
    }
}

Sliding Window Median (LeetCode #480) ▼

Problem: Given an array of integers and a window size k, find the median of each window as it slides from left to right.

Key Challenge: Unlike the stream problem, we also need to remove elements as the window moves. Heaps do not natively support removal by value efficiently, so we use lazy deletion: mark elements as removed and clean them up when they appear at the top.

import heapq
from collections import defaultdict

def medianSlidingWindow(nums, k):
    """
    LeetCode #480 - Sliding Window Median

    Two heaps + lazy deletion using a balance counter.
    Time: O(n log k), Space: O(k)
    """
    small = []  # max-heap (negated)
    large = []  # min-heap
    delayed = defaultdict(int)  # element -> count of pending removals

    # small_size and large_size track "logical" sizes (excluding lazy-deleted)
    small_size = 0
    large_size = 0
    result = []

    def prune(heap):
        """Remove elements from top that are marked for lazy deletion."""
        while heap:
            val = -heap[0] if heap is small else heap[0]
            if delayed[val] > 0:
                delayed[val] -= 1
                heapq.heappop(heap)
            else:
                break

    def rebalance():
        """Ensure small_size == large_size or small_size == large_size + 1."""
        nonlocal small_size, large_size
        if small_size > large_size + 1:
            # Move top of small to large
            val = -heapq.heappop(small)
            heapq.heappush(large, val)
            small_size -= 1
            large_size += 1
            prune(small)
        elif large_size > small_size:
            val = heapq.heappop(large)
            heapq.heappush(small, -val)
            large_size -= 1
            small_size += 1
            prune(large)

    def add(num):
        nonlocal small_size, large_size
        if not small or num <= -small[0]:
            heapq.heappush(small, -num)
            small_size += 1
        else:
            heapq.heappush(large, num)
            large_size += 1
        rebalance()

    def remove(num):
        nonlocal small_size, large_size
        delayed[num] += 1
        if num <= -small[0]:
            small_size -= 1
        else:
            large_size -= 1
        rebalance()
        prune(small)
        prune(large)

    def get_median():
        if k % 2 == 1:
            return float(-small[0])
        return (-small[0] + large[0]) / 2.0

    # Initialize first window
    for i in range(k):
        add(nums[i])
    result.append(get_median())

    # Slide the window
    for i in range(k, len(nums)):
        add(nums[i])
        remove(nums[i - k])
        result.append(get_median())

    return result


# Example:
# nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3
# Output: [1.0, -1.0, -1.0, 3.0, 5.0, 6.0]
IPO Problem (LeetCode #502) ▼

Problem: You have initial capital W. Each project i has a profit[i] and minimum capital[i] required. You can do at most k projects. After completing a project, your capital increases by its profit. Maximize your final capital.

Key Insight: Use a min-heap to track projects by required capital (to quickly find affordable ones), and a max-heap to pick the most profitable affordable project.

import heapq

def findMaximizedCapital(k, w, profits, capital):
    """
    LeetCode #502 - IPO

    Two heaps:
      - min-heap sorted by capital requirement (what we can afford)
      - max-heap sorted by profit (best available project)

    Time: O(n log n + k log n), Space: O(n)
    """
    # Min-heap of (capital_required, profit)
    min_cap = [(c, p) for c, p in zip(capital, profits)]
    heapq.heapify(min_cap)

    # Max-heap of profits (for affordable projects)
    max_profit = []

    for _ in range(k):
        # Move all affordable projects to the max-profit heap
        while min_cap and min_cap[0][0] <= w:
            c, p = heapq.heappop(min_cap)
            heapq.heappush(max_profit, -p)

        # Pick the most profitable affordable project
        if not max_profit:
            break  # No affordable projects left

        w += -heapq.heappop(max_profit)

    return w


# Example:
# k=2, w=0, profits=[1,2,3], capital=[0,1,1]
# Round 1: afford project 0 (cap=0), profit=1, w=1
# Round 2: afford projects 1,2 (cap=1), pick profit=3, w=4
# Output: 4
Two Heaps Pattern Template ▼

When to Use Two Heaps

  • You need the median of a stream or window
  • You need to partition data into two halves and efficiently query the boundary
  • You need the best option from a dynamically changing set (like IPO)
  • Problems involving "maximize minimum" or "minimize maximum"

Template

import heapq

class TwoHeapTemplate:
    """
    Generic Two Heaps pattern:
    - max_heap (small): stores elements in lower partition (negate for Python)
    - min_heap (large): stores elements in upper partition

    Invariant: all elements in small <= all elements in large
    Balance:   |len(small) - len(large)| <= 1
    """
    def __init__(self):
        self.small = []  # max-heap (negated)
        self.large = []  # min-heap

    def add(self, num):
        # 1. Add to appropriate heap
        if not self.small or num <= -self.small[0]:
            heapq.heappush(self.small, -num)
        else:
            heapq.heappush(self.large, num)

        # 2. Rebalance
        if len(self.small) > len(self.large) + 1:
            heapq.heappush(self.large, -heapq.heappop(self.small))
        elif len(self.large) > len(self.small):
            heapq.heappush(self.small, -heapq.heappop(self.large))

    def get_boundary(self):
        """Returns the median or partition boundary."""
        if len(self.small) > len(self.large):
            return -self.small[0]
        return (-self.small[0] + self.large[0]) / 2.0

K-Way Merge Pattern

K-Way Merge: The Distributed Systems Workhorse

Whenever you have K sorted sources and need to produce a single sorted output, the K-way merge pattern using a min-heap of size K is the go-to approach. It powers external sorting, merging database SSTables, Kafka log compaction, and distributed search ranking.

Merge K Sorted Lists (LeetCode #23) - Full Solution ▼

Problem: Merge k sorted linked lists into one sorted linked list.

Why this matters: This is the #1 most-asked heap problem at FAANG. It tests your ability to use heaps for merge operations and handle edge cases with linked lists.

Approach Comparison

Approach Time Space When to Use
Brute force (collect all, sort)O(N log N)O(N)Only when k is very large
Heap (min-heap of size k)O(N log k)O(k)Best general approach
Divide and conquer mergeO(N log k)O(log k)Recursive elegance, less space
 
import heapq
from typing import List, Optional

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def mergeKLists(lists: List[Optional[ListNode]]) -> Optional[ListNode]:
    """
    LeetCode #23 - Merge K Sorted Lists (Heap Approach)

    Use a min-heap of size k. Each entry is (node.val, index, node).
    The index is a tiebreaker since ListNode is not comparable.

    Time: O(N log k) where N = total nodes across all lists
    Space: O(k) for the heap
    """
    heap = []

    # Initialize: push first node from each list
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    dummy = ListNode(0)
    curr = dummy

    while heap:
        val, idx, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next

        if node.next:
            heapq.heappush(heap, (node.next.val, idx, node.next))

    return dummy.next


def mergeKLists_divideconquer(lists: List[Optional[ListNode]]) -> Optional[ListNode]:
    """
    Divide and Conquer approach.
    Pair up lists and merge them, reducing k by half each round.

    Time: O(N log k), Space: O(log k) recursion stack
    """
    if not lists:
        return None

    def merge2(l1, l2):
        dummy = ListNode(0)
        curr = dummy
        while l1 and l2:
            if l1.val <= l2.val:
                curr.next = l1
                l1 = l1.next
            else:
                curr.next = l2
                l2 = l2.next
            curr = curr.next
        curr.next = l1 or l2
        return dummy.next

    # Iteratively merge pairs
    while len(lists) > 1:
        merged = []
        for i in range(0, len(lists), 2):
            l1 = lists[i]
            l2 = lists[i + 1] if i + 1 < len(lists) else None
            merged.append(merge2(l1, l2))
        lists = merged

    return lists[0]
Smallest Range Covering Elements from K Lists (LeetCode #632) ▼

Problem: Given k sorted lists, find the smallest range [a, b] that includes at least one number from each list.

Key Insight: Use a min-heap to track one element from each list. The range at any point is [heap_min, current_max]. Advance the minimum element to try to shrink the range.

import heapq
from typing import List

def smallestRange(nums: List[List[int]]) -> List[int]:
    """
    LeetCode #632 - Smallest Range Covering Elements from K Lists

    Maintain a min-heap with one element from each list.
    Track the current maximum. Range = [heap_min, current_max].
    Pop the minimum and push its next element. Update range if smaller.

    Time: O(N log k) where N = total elements
    Space: O(k)
    """
    # heap entries: (value, list_index, element_index)
    heap = []
    current_max = float('-inf')

    # Initialize with first element from each list
    for i, lst in enumerate(nums):
        heapq.heappush(heap, (lst[0], i, 0))
        current_max = max(current_max, lst[0])

    best_range = [heap[0][0], current_max]

    while True:
        min_val, list_idx, elem_idx = heapq.heappop(heap)

        # Update best range if current is smaller
        if current_max - min_val < best_range[1] - best_range[0]:
            best_range = [min_val, current_max]

        # Move to next element in the list that had the minimum
        next_idx = elem_idx + 1
        if next_idx >= len(nums[list_idx]):
            break  # One list exhausted - can't cover all lists anymore

        next_val = nums[list_idx][next_idx]
        heapq.heappush(heap, (next_val, list_idx, next_idx))
        current_max = max(current_max, next_val)

    return best_range


# Example:
# nums = [[4,10,15,24,26], [0,9,12,20], [5,18,22,30]]
# Output: [20, 24] -- covers 24 from list 0, 20 from list 1, 22 from list 2
Kth Smallest Element in Sorted Matrix (LeetCode #378) ▼

Problem: Given an n x n matrix where each row and column is sorted in ascending order, find the kth smallest element.

Two approaches: K-way merge with heap (treat each row as a sorted list) or binary search on the value range.

import heapq
from typing import List

def kthSmallest_heap(matrix: List[List[int]], k: int) -> int:
    """
    Approach 1: K-Way Merge with Heap
    Treat each row as a sorted list. Pop the smallest k times.

    Time: O(k log n), Space: O(n)
    """
    n = len(matrix)
    # heap: (value, row, col)
    heap = [(matrix[i][0], i, 0) for i in range(n)]
    heapq.heapify(heap)

    for _ in range(k - 1):
        val, r, c = heapq.heappop(heap)
        if c + 1 < n:
            heapq.heappush(heap, (matrix[r][c + 1], r, c + 1))

    return heap[0][0]


def kthSmallest_binarysearch(matrix: List[List[int]], k: int) -> int:
    """
    Approach 2: Binary Search on Value
    Binary search between matrix[0][0] and matrix[n-1][n-1].
    For each mid value, count how many elements <= mid.

    Time: O(n log(max - min)), Space: O(1)
    """
    n = len(matrix)
    lo, hi = matrix[0][0], matrix[n - 1][n - 1]

    def count_le(target):
        """Count elements <= target using staircase walk. O(n)."""
        count = 0
        row, col = n - 1, 0  # Start from bottom-left
        while row >= 0 and col < n:
            if matrix[row][col] <= target:
                count += row + 1  # All elements above are also <= target
                col += 1
            else:
                row -= 1
        return count

    while lo < hi:
        mid = lo + (hi - lo) // 2
        if count_le(mid) < k:
            lo = mid + 1
        else:
            hi = mid

    return lo


# Example:
# matrix = [[1,5,9],[10,11,13],[12,13,15]], k = 8
# Output: 13

Sorting Deep Dive: Beyond Comparison Sorts

O(n) Sorts: When to Break the O(n log n) Barrier

Comparison-based sorts have a theoretical lower bound of O(n log n). But when your data has special properties (bounded integers, uniform distribution), counting sort, radix sort, and bucket sort achieve O(n) time. Knowing when to use them is a staff-level differentiator.

Counting Sort, Radix Sort, Bucket Sort ▼

Counting Sort

When it shines: Small, bounded integer range (e.g., ages 0-150, ASCII chars 0-127, scores 0-100). It counts occurrences rather than comparing elements.

def counting_sort(arr, max_val=None):
    """
    Counting Sort - O(n + k) time, O(k) space
    where k = range of values.

    Stable: Yes (preserves relative order of equal elements)
    Not comparison-based: bypasses O(n log n) lower bound.

    Best for: small integer range (k << n or k ~ n)
    """
    if not arr:
        return arr

    if max_val is None:
        max_val = max(arr)

    # Count occurrences
    count = [0] * (max_val + 1)
    for num in arr:
        count[num] += 1

    # Build sorted output
    result = []
    for val in range(max_val + 1):
        result.extend([val] * count[val])

    return result


def counting_sort_stable(arr, key_func):
    """
    Stable counting sort using cumulative counts.
    Essential for radix sort (which needs stability at each digit).
    """
    if not arr:
        return arr

    max_key = max(key_func(x) for x in arr)
    count = [0] * (max_key + 1)

    for x in arr:
        count[key_func(x)] += 1

    # Cumulative count gives final positions
    for i in range(1, len(count)):
        count[i] += count[i - 1]

    # Build output (traverse right-to-left for stability)
    output = [None] * len(arr)
    for x in reversed(arr):
        k = key_func(x)
        count[k] -= 1
        output[count[k]] = x

    return output

Radix Sort

When it shines: Sorting integers or fixed-length strings. Processes digits from least significant to most significant, using counting sort as a subroutine at each digit. Runs in O(d * (n + b)) where d = number of digits and b = base.

def radix_sort(arr):
    """
    Radix Sort (LSD - Least Significant Digit first)
    Time: O(d * (n + b)) where d = digits, b = base (10)
    Space: O(n + b)

    Best for: integers with bounded # of digits.
    Example: sorting 1M 32-bit integers = O(4 * n) with base 256.
    """
    if not arr:
        return arr

    max_val = max(arr)
    exp = 1  # Current digit position (1s, 10s, 100s, ...)

    while max_val // exp > 0:
        # Counting sort by current digit
        arr = _counting_sort_by_digit(arr, exp)
        exp *= 10

    return arr

def _counting_sort_by_digit(arr, exp):
    """Stable sort by the digit at position exp."""
    n = len(arr)
    output = [0] * n
    count = [0] * 10

    for num in arr:
        digit = (num // exp) % 10
        count[digit] += 1

    for i in range(1, 10):
        count[i] += count[i - 1]

    for i in range(n - 1, -1, -1):
        digit = (arr[i] // exp) % 10
        count[digit] -= 1
        output[count[digit]] = arr[i]

    return output


# For negative numbers, separate into negatives and positives,
# radix sort each, then concatenate.

Bucket Sort

When it shines: Uniformly distributed floating-point numbers. Distributes elements into buckets, sorts each bucket (using insertion sort for small buckets), then concatenates.

def bucket_sort(arr):
    """
    Bucket Sort
    Time: O(n) average when data is uniformly distributed
    Worst case: O(n^2) when all elements land in one bucket
    Space: O(n)

    Best for: uniformly distributed floats in [0, 1)
    """
    if not arr:
        return arr

    n = len(arr)
    buckets = [[] for _ in range(n)]

    # Distribute elements into buckets
    for num in arr:
        bucket_idx = int(n * num)  # For [0, 1) range
        bucket_idx = min(bucket_idx, n - 1)  # Handle edge case
        buckets[bucket_idx].append(num)

    # Sort individual buckets (insertion sort for small buckets)
    for bucket in buckets:
        bucket.sort()  # Python's Timsort is efficient for small arrays

    # Concatenate all buckets
    result = []
    for bucket in buckets:
        result.extend(bucket)

    return result


# Real-world: used in histogram equalization, geographic binning,
# and database query optimization for range queries.

Tim Sort: What Python and Java Actually Use ▼

Tim Sort is the default sorting algorithm in Python (list.sort(), sorted()), Java (Arrays.sort() for objects), and many other languages. Invented by Tim Peters in 2002, it is a hybrid of merge sort and insertion sort, optimized for real-world data that often contains partially ordered subsequences ("runs").

How Tim Sort Works

  1. Find natural runs: Scan the array for already-sorted subsequences (ascending or descending). Reverse descending runs in-place.
  2. Extend short runs: If a run is shorter than minrun (typically 32-64), extend it using insertion sort. Insertion sort is extremely efficient for small arrays due to low overhead and cache locality.
  3. Merge runs: Push runs onto a stack and merge them using a modified merge sort that maintains a merging invariant (similar to the Fibonacci sequence) to ensure balanced merges.
  4. Galloping mode: When one run consistently "wins" during merge (contributes many elements in a row), switch to binary search ("galloping") to find the insertion point, skipping large chunks. This makes merging nearly-sorted data very fast.

Stability: Why It Matters

A stable sort preserves the relative order of elements with equal keys. This is critical when:

  • Multi-key sorting: Sort by last name, then by first name. Stability ensures the first-name order is preserved within same last names.
  • Database queries: ORDER BY date, priority requires stability to respect the secondary sort.
  • Radix sort correctness: Each digit pass must be stable, or previous digit orderings are destroyed.
  • UI rendering: Sorting a table by one column should not randomly rearrange rows with equal values in that column.

import heapq
import random

# ---- Approach 1: Min-Heap (preferred for interviews) ----
def findKthLargest_heap(nums, k):
    """
    Min-heap of size k. After processing all elements,
    the heap top is the kth largest.

    Time: O(n log k), Space: O(k)
    Advantage: Works for streams, doesn't modify input.
    """
    heap = []
    for num in nums:
        if len(heap) < k:
            heapq.heappush(heap, num)
        elif num > heap[0]:
            heapq.heapreplace(heap, num)  # More efficient than pop+push
    return heap[0]


# ---- Approach 2: QuickSelect (optimal time) ----
def findKthLargest_quickselect(nums, k):
    """
    QuickSelect: partition-based selection.
    Average O(n), worst O(n^2). Randomize pivot to avoid worst case.

    Time: O(n) average, Space: O(1)
    Advantage: Fastest single-pass for static arrays.
    Disadvantage: Mutates input, O(n^2) worst case.
    """
    k = len(nums) - k  # Convert to index of kth smallest

    def quickselect(lo, hi):
        # Randomize pivot
        pivot_idx = random.randint(lo, hi)
        nums[pivot_idx], nums[hi] = nums[hi], nums[pivot_idx]

        pivot = nums[hi]
        store = lo

        for i in range(lo, hi):
            if nums[i] <= pivot:
                nums[store], nums[i] = nums[i], nums[store]
                store += 1

        nums[store], nums[hi] = nums[hi], nums[store]

        if store == k:
            return nums[store]
        elif store < k:
            return quickselect(store + 1, hi)
        else:
            return quickselect(lo, store - 1)

    return quickselect(0, len(nums) - 1)


# ---- Approach 3: Bucket Sort (when values are bounded) ----
def findKthLargest_bucket(nums, k):
    """
    When values are bounded, use counting/bucket approach.
    Time: O(n + range), Space: O(range)

    Example: LeetCode constrains values to [-10^4, 10^4]
    """
    min_val = min(nums)
    max_val = max(nums)
    count = [0] * (max_val - min_val + 1)

    for num in nums:
        count[num - min_val] += 1

    # Walk from largest to smallest
    remaining = k
    for val in range(max_val, min_val - 1, -1):
        remaining -= count[val - min_val]
        if remaining <= 0:
            return val

    return min_val  # Should not reach here
Top K Frequent Elements (LeetCode #347) - Heap vs QuickSelect vs Bucket Sort ▼

Problem: Given an integer array and k, return the k most frequent elements. The answer is guaranteed to be unique.

This problem is a favorite because it can be solved three completely different ways, and the interviewer wants to see you reason about which is best and why.

from collections import Counter
import heapq

# ---- Approach 1: Heap - O(n log k) ----
def topKFrequent_heap(nums, k):
    """
    Count frequencies, then use min-heap of size k
    to find k most frequent.

    Time: O(n + m log k) where m = unique elements
    Space: O(m + k)
    """
    freq = Counter(nums)

    # Min-heap of (frequency, element)
    heap = []
    for num, count in freq.items():
        if len(heap) < k:
            heapq.heappush(heap, (count, num))
        elif count > heap[0][0]:
            heapq.heapreplace(heap, (count, num))

    return [num for _, num in heap]


# ---- Approach 2: Bucket Sort - O(n) ----
def topKFrequent_bucket(nums, k):
    """
    Bucket sort by frequency. Index = frequency, value = list of elements.
    Maximum frequency is n, so bucket array has size n+1.

    Time: O(n), Space: O(n)
    This is the OPTIMAL approach for this problem.
    """
    freq = Counter(nums)
    n = len(nums)

    # Buckets: index = frequency, value = elements with that frequency
    buckets = [[] for _ in range(n + 1)]
    for num, count in freq.items():
        buckets[count].append(num)

    # Collect from highest frequency bucket
    result = []
    for i in range(n, 0, -1):
        for num in buckets[i]:
            result.append(num)
            if len(result) == k:
                return result

    return result


# ---- Approach 3: QuickSelect on frequencies - O(n) avg ----
def topKFrequent_quickselect(nums, k):
    """
    Count frequencies, then use QuickSelect to find
    the kth most frequent element threshold.

    Time: O(n) average, Space: O(n)
    """
    freq = Counter(nums)
    unique = list(freq.keys())

    def partition(lo, hi):
        pivot_freq = freq[unique[hi]]
        store = lo
        for i in range(lo, hi):
            if freq[unique[i]] >= pivot_freq:
                unique[store], unique[i] = unique[i], unique[store]
                store += 1
        unique[store], unique[hi] = unique[hi], unique[store]
        return store

    def quickselect(lo, hi):
        if lo == hi:
            return
        pivot_idx = partition(lo, hi)
        if pivot_idx == k:
            return
        elif pivot_idx < k:
            quickselect(pivot_idx + 1, hi)
        else:
            quickselect(lo, pivot_idx - 1)

    quickselect(0, len(unique) - 1)
    return unique[:k]

# Kafka-style log compaction using heap
# Keeps only latest value per key to reduce storage

import heapq
from collections import defaultdict

class LogCompactor:
    """
    Kafka log compaction: retain only latest value per key.
    Scale: Kafka compacts billions of records to save 70%+ storage.

    Uses heap to process records in timestamp order,
    keeping only the most recent per key.
    """

    def compact(self, log_segments):
        """
        Input: Multiple log segments, each with [(key, value, timestamp), ...]
        Output: Compacted log with only latest value per key

        Algorithm: Merge segments using heap, track latest per key
        Time: O(N log K) where N = total records, K = segments
        """
        # Track latest value per key
        key_values = {}  # key -> (value, timestamp)

        # Min-heap for k-way merge: (timestamp, segment_idx, record, iterator)
        heap = []
        iterators = []

        # Initialize heap with first record from each segment
        for seg_idx, segment in enumerate(log_segments):
            it = iter(sorted(segment, key=lambda x: x[2]))  # Sort by timestamp
            iterators.append(it)
            try:
                record = next(it)
                heapq.heappush(heap, (record[2], seg_idx, record))
            except StopIteration:
                pass

        # Process all records in timestamp order
        while heap:
            timestamp, seg_idx, record = heapq.heappop(heap)
            key, value, ts = record

            # Update if this is newer (timestamp ordering ensures it is)
            if value is not None:  # None = tombstone (delete marker)
                key_values[key] = (value, ts)
            elif key in key_values:
                del key_values[key]  # Key deleted

            # Get next record from same segment
            try:
                next_record = next(iterators[seg_idx])
                heapq.heappush(heap, (next_record[2], seg_idx, next_record))
            except StopIteration:
                pass

        # Return compacted log
        return [(k, v, ts) for k, (v, ts) in key_values.items()]


class DelayedMessageQueue:
    """
    Kafka delayed delivery: schedule messages for future timestamps.
    Used for: retry after backoff, scheduled notifications, rate limiting.

    Priority queue (min-heap) by delivery timestamp.
    """

    def __init__(self):
        self.heap = []  # (delivery_time, sequence, message)
        self.sequence = 0  # Tie-breaker for same timestamp

    def schedule(self, message, deliver_at):
        """Add message to be delivered at specific time."""
        heapq.heappush(self.heap, (deliver_at, self.sequence, message))
        self.sequence += 1

    def get_ready_messages(self, current_time):
        """Get all messages ready for delivery."""
        ready = []
        while self.heap and self.heap[0][0] <= current_time:
            _, _, message = heapq.heappop(self.heap)
            ready.append(message)
        return ready

    def next_delivery_time(self):
        """When is the next message due?"""
        return self.heap[0][0] if self.heap else None

High-Frequency Trading: Real-Time Order Book (Microsecond Latency) ▼

Scale: NYSE processes 6+ billion messages per day. HFT systems must maintain sorted order books and find best prices in nanoseconds.

Order Book = Two Heaps

An order book tracks all buy and sell orders. Matching engine needs instant access to best bid (max-heap) and best ask (min-heap).

# HFT Order Book using heaps
# Real systems use custom data structures for nanosecond performance

import heapq
from collections import defaultdict
from dataclasses import dataclass
from typing import List, Optional
import time

@dataclass
class Order:
    order_id: str
    price: int        # Price in cents to avoid float issues
    quantity: int
    timestamp: float  # Nanosecond precision in real systems
    side: str         # 'BUY' or 'SELL'

class OrderBook:
    """
    Matching engine order book using heaps.
    Scale: NASDAQ processes 1M+ orders/second with <10μs latency.

    Buy side: Max-heap (best bid = highest price)
    Sell side: Min-heap (best ask = lowest price)
    """

    def __init__(self, symbol: str):
        self.symbol = symbol
        self.buy_heap = []   # Max-heap: (-price, timestamp, order)
        self.sell_heap = []  # Min-heap: (price, timestamp, order)
        self.orders = {}     # order_id -> order (for cancellation)

    def add_order(self, order: Order) -> List[dict]:
        """
        Add order and attempt to match.
        Returns list of executed trades.

        Time: O(log n) for add, O(m log n) for m matches
        """
        trades = []

        if order.side == 'BUY':
            # Try to match with sell orders
            while (order.quantity > 0 and
                   self.sell_heap and
                   self.sell_heap[0][0] <= order.price):

                sell_price, _, sell_order = self.sell_heap[0]

                # Execute trade at sell price (price-time priority)
                trade_qty = min(order.quantity, sell_order.quantity)
                trades.append({
                    'price': sell_price,
                    'quantity': trade_qty,
                    'buyer': order.order_id,
                    'seller': sell_order.order_id,
                    'timestamp': time.time_ns()
                })

                order.quantity -= trade_qty
                sell_order.quantity -= trade_qty

                if sell_order.quantity == 0:
                    heapq.heappop(self.sell_heap)
                    del self.orders[sell_order.order_id]

            # Add remaining to book
            if order.quantity > 0:
                heapq.heappush(self.buy_heap,
                    (-order.price, order.timestamp, order))
                self.orders[order.order_id] = order

        else:  # SELL
            # Try to match with buy orders
            while (order.quantity > 0 and
                   self.buy_heap and
                   -self.buy_heap[0][0] >= order.price):

                neg_buy_price, _, buy_order = self.buy_heap[0]
                buy_price = -neg_buy_price

                trade_qty = min(order.quantity, buy_order.quantity)
                trades.append({
                    'price': buy_price,  # Execute at buy price
                    'quantity': trade_qty,
                    'buyer': buy_order.order_id,
                    'seller': order.order_id,
                    'timestamp': time.time_ns()
                })

                order.quantity -= trade_qty
                buy_order.quantity -= trade_qty

                if buy_order.quantity == 0:
                    heapq.heappop(self.buy_heap)
                    del self.orders[buy_order.order_id]

            if order.quantity > 0:
                heapq.heappush(self.sell_heap,
                    (order.price, order.timestamp, order))
                self.orders[order.order_id] = order

        return trades

    def get_best_bid(self) -> Optional[float]:
        """Best buy price - O(1)."""
        while self.buy_heap and self.buy_heap[0][2].quantity == 0:
            heapq.heappop(self.buy_heap)
        return -self.buy_heap[0][0] if self.buy_heap else None

    def get_best_ask(self) -> Optional[float]:
        """Best sell price - O(1)."""
        while self.sell_heap and self.sell_heap[0][2].quantity == 0:
            heapq.heappop(self.sell_heap)
        return self.sell_heap[0][0] if self.sell_heap else None

    def get_spread(self) -> Optional[float]:
        """Bid-ask spread - key market metric."""
        bid, ask = self.get_best_bid(), self.get_best_ask()
        return (ask - bid) if bid and ask else None

Uber Surge Pricing: Real-Time Demand Aggregation ▼

Scale: Uber processes roughly 30 million trips daily across 10,000+ cities. Surge pricing updates every few minutes based on supply/demand in each area.

The Problem: Aggregate Demand in Grid Cells

Uber divides cities into hexagonal cells (H3 grid). Each cell needs real-time demand metrics to calculate surge. This is a streaming Top-K problem.

# Uber-style surge pricing using heap for top demand areas
# Simplied version of their H3-based geospatial system

import heapq
from collections import defaultdict
from dataclasses import dataclass
import time

@dataclass
class RideRequest:
    cell_id: str      # H3 hexagon ID
    timestamp: float
    rider_id: str

class SurgePricingEngine:
    """
    Real-time demand aggregation for surge pricing.
    Scale: Uber aggregates millions of events per minute.

    Uses sliding window + heap to find top demand areas.
    """

    def __init__(self, window_seconds=300, top_k=100):
        self.window = window_seconds  # 5-minute window
        self.top_k = top_k
        self.cell_requests = defaultdict(list)  # cell -> [timestamps]
        self.supply = defaultdict(int)  # cell -> available drivers

    def record_request(self, request: RideRequest):
        """Record ride request - O(1)."""
        self.cell_requests[request.cell_id].append(request.timestamp)

    def get_surge_areas(self) -> list:
        """
        Get top-K highest demand areas.
        Uses min-heap to efficiently track top demand cells.

        Time: O(C log K) where C = number of cells
        """
        current_time = time.time()
        cutoff = current_time - self.window

        # Calculate demand for each cell (requests in window)
        demand_scores = []
        for cell_id, timestamps in self.cell_requests.items():
            # Remove old requests (sliding window)
            self.cell_requests[cell_id] = [
                ts for ts in timestamps if ts > cutoff
            ]

            recent_requests = len(self.cell_requests[cell_id])
            drivers = self.supply.get(cell_id, 1)

            # Demand score = requests / supply
            # High score = need surge pricing
            score = recent_requests / max(drivers, 1)
            demand_scores.append((cell_id, score, recent_requests))

        # Use min-heap of size K to find top-K
        # O(n log k) vs O(n log n) for full sort
        # Note: CPython's nlargest falls back to sorted() when k is large relative to n
        top_cells = heapq.nlargest(
            self.top_k,
            demand_scores,
            key=lambda x: x[1]
        )

        # Calculate surge multiplier
        result = []
        for cell_id, score, requests in top_cells:
            # Surge multiplier based on demand/supply ratio
            # Real Uber uses ML models with many more factors
            surge = min(1 + (score - 1) * 0.5, 3.0)  # Cap at 3x
            surge = max(surge, 1.0)  # Min 1x

            result.append({
                'cell_id': cell_id,
                'demand_score': score,
                'surge_multiplier': round(surge, 2),
                'requests_in_window': requests
            })

        return result

    def update_supply(self, cell_id: str, drivers: int):
        """Update available drivers in cell."""
        self.supply[cell_id] = drivers


class StreamingTopK:
    """
    Maintain streaming top-K with heap.
    Uses lazy deletion: stale entries stay in heap until popped.
    For exact results, call get_top_k() which filters via self.scores.
    """

    def __init__(self, k: int):
        self.k = k
        self.heap = []  # Min-heap of (score, id) - may contain stale entries
        self.scores = {}  # id -> current score (source of truth)

    def update(self, item_id: str, score: float):
        """
        Update item's score. Maintains top-K approximately.
        Stale heap entries are cleaned up lazily in get_top_k().
        Time: O(log K) amortized
        """
        self.scores[item_id] = score

        if len(self.heap) < self.k:
            heapq.heappush(self.heap, (score, item_id))
        elif score > self.heap[0][0]:
            heapq.heappush(self.heap, (score, item_id))

    def get_top_k(self):
        """Return current top-K, filtering stale entries. O(K log K)."""
        # Filter to entries whose heap score matches current score
        valid = [(s, id) for s, id in self.heap if self.scores.get(id) == s]
        return sorted(valid, reverse=True)[:self.k]
Google Search: Sorting Billions of Results ▼

Google processes 8.5 billion searches per day. Each query matches millions of documents that must be ranked and sorted. But users only see the top 10 - this is a Top-K problem at massive scale.

Why Full Sorting is Impossible

  • Query "python": Matches ~1 billion documents
  • Full sort: O(n log n) = 30 billion comparisons
  • Top-K with heap: O(n log k) = 10 billion comparisons for top 1000
  • Reality: Google uses multi-stage filtering + heap merge
# Google-style distributed top-K search results
import heapq

class DistributedSearchRanker:
    """
    Search results come from 1000s of index shards.
    Each shard returns its local top-K, then we merge.
    """
    def __init__(self, num_shards=1000, local_k=100, global_k=10):
        self.num_shards = num_shards
        self.local_k = local_k
        self.global_k = global_k

    def search_shard(self, shard_id, query):
        """
        Each shard has ~1M documents.
        Returns top local_k results using heap.
        """
        # In reality: TF-IDF / BM25 scoring + PageRank
        results = self.score_documents(shard_id, query)

        # Use min-heap of size k to find top-k
        # O(n log k) where n = documents in shard
        top_k = heapq.nlargest(self.local_k, results, key=lambda x: x['score'])
        return top_k

    def merge_results(self, shard_results):
        """
        Merge top-k from each shard into global top-k.
        This is k-way merge with heap: O(shards * local_k * log(shards))
        """
        # Create iterator for each shard's results
        iterators = [iter(sorted(r, key=lambda x: -x['score']))
                    for r in shard_results]

        # Min heap of (negative_score, shard_index, doc)
        heap = []
        for i, it in enumerate(iterators):
            try:
                doc = next(it)
                heapq.heappush(heap, (-doc['score'], i, doc, it))
            except StopIteration:
                pass

        # Extract global top-k
        global_top_k = []
        while heap and len(global_top_k) < self.global_k:
            neg_score, shard_idx, doc, it = heapq.heappop(heap)
            global_top_k.append(doc)

            try:
                next_doc = next(it)
                heapq.heappush(heap, (-next_doc['score'], shard_idx, next_doc, it))
            except StopIteration:
                pass

        return global_top_k

# Phase 1: Fast scorer returns top 1000 (approximate)
# Phase 2: Expensive ML model re-ranks top 1000 precisely
Twitter: Real-Time Timeline Ranking with Heaps ▼

X (formerly Twitter) has historically processed around 500 million posts per day. Your timeline merges posts from 1000s of accounts you follow, ranked by relevance - a classic k-way merge problem.

Fan-out on Read vs Write

# Twitter-style timeline generation
import heapq
from collections import defaultdict

class TimelineService:
    """
    Two approaches to timeline:
    1. Fan-out on write: Pre-compute timelines (fast read, slow write)
    2. Fan-out on read: Compute on demand (slow read, fast write)

    Twitter uses hybrid: celebrities fan-out on read, others on write.
    """
    def __init__(self):
        self.user_tweets = defaultdict(list)  # user -> [tweets]
        self.followers = defaultdict(set)      # user -> {followers}
        self.precomputed_timelines = {}        # For non-celebrity users

    def post_tweet(self, user_id, tweet):
        """O(1) for celebrities, O(followers) for regular users."""
        self.user_tweets[user_id].append(tweet)

        if self.is_celebrity(user_id):
            # Don't fan out - too many followers
            pass
        else:
            # Fan-out on write to followers' precomputed timelines
            for follower in self.followers[user_id]:
                if follower in self.precomputed_timelines:
                    self._add_to_timeline(follower, tweet)

    def get_timeline(self, user_id, limit=20):
        """
        Merge tweets from followed users using heap.
        O(following * tweets_per_user * log(following))
        """
        following = self.get_following(user_id)

        # Create heap of (timestamp, tweet, user_iterator)
        heap = []
        iterators = {}

        for followed_user in following:
            tweets = self.user_tweets[followed_user]
            if tweets:
                # Iterator from newest to oldest
                it = iter(reversed(tweets))
                tweet = next(it)
                iterators[followed_user] = it
                # Use negative timestamp for max-heap behavior
                heapq.heappush(heap, (-tweet['timestamp'], followed_user, tweet))

        # Extract top 'limit' tweets
        timeline = []
        while heap and len(timeline) < limit:
            _, user, tweet = heapq.heappop(heap)
            timeline.append(tweet)

            it = iterators.get(user)
            if it:
                try:
                    next_tweet = next(it)
                    heapq.heappush(heap, (-next_tweet['timestamp'], user, next_tweet))
                except StopIteration:
                    pass

        return timeline

Apache Spark: External Sorting for Big Data ▼

When data exceeds memory, external sorting becomes essential. Apache Spark, used by Netflix, Uber, and Airbnb, implements sophisticated external merge sort.

External Sort Algorithm

# External merge sort (basis of Spark's shuffle)
import heapq
import tempfile
import os

class ExternalSorter:
    """
    Sort data larger than memory.
    Used in: Spark shuffle, database ORDER BY, MapReduce.
    """
    def __init__(self, memory_limit_mb=100):
        self.memory_limit = memory_limit_mb * 1024 * 1024
        self.temp_files = []

    def sort(self, input_file, output_file, key_func=lambda x: x):
        """
        Phase 1: Create sorted runs that fit in memory
        Phase 2: K-way merge sorted runs

        I/O complexity: O(N/M * log(N/M)) disk reads
        where N = data size, M = memory size
        """
        # Phase 1: Create sorted runs
        self._create_sorted_runs(input_file, key_func)

        # Phase 2: K-way merge
        self._merge_runs(output_file, key_func)

        # Cleanup temp files
        for f in self.temp_files:
            os.unlink(f)

    def _create_sorted_runs(self, input_file, key_func):
        """Load chunks, sort in memory, write to temp files."""
        chunk = []
        chunk_size = 0

        with open(input_file, 'r') as f:
            for line in f:
                chunk.append(line)
                chunk_size += len(line)

                if chunk_size >= self.memory_limit:
                    self._write_sorted_run(chunk, key_func)
                    chunk = []
                    chunk_size = 0

            if chunk:
                self._write_sorted_run(chunk, key_func)

    def _write_sorted_run(self, chunk, key_func):
        """Sort chunk and write to temp file."""
        chunk.sort(key=key_func)

        temp_fd, temp_path = tempfile.mkstemp()
        with os.fdopen(temp_fd, 'w') as f:
            for line in chunk:
                f.write(line)

        self.temp_files.append(temp_path)

    def _merge_runs(self, output_file, key_func):
        """K-way merge using min-heap."""
        # Open all temp files
        file_handles = [open(f, 'r') for f in self.temp_files]

        # Initialize heap with first element from each file
        heap = []
        for i, fh in enumerate(file_handles):
            line = fh.readline()
            if line:
                heapq.heappush(heap, (key_func(line), i, line))

        # Merge
        with open(output_file, 'w') as out:
            while heap:
                _, file_idx, line = heapq.heappop(heap)
                out.write(line)

                next_line = file_handles[file_idx].readline()
                if next_line:
                    heapq.heappush(heap, (key_func(next_line), file_idx, next_line))

        # Close file handles
        for fh in file_handles:
            fh.close()

Self-Check Quiz

Interview Follow-up Questions

Common Follow-ups & What Interviewers Look For ▼

1. "Why use a min-heap for finding top-K largest elements?"

What they're looking for:

  • Min-heap of size K: smallest element is the "gatekeeper"
  • If new element > min, pop min and push new element
  • O(n log k) vs O(n log n) for full sort

Red flags: Using max-heap and keeping all n elements.

2. "When would you use QuickSelect instead of a heap for top-K?"

What they're looking for:

  • QuickSelect: O(n) average but modifies input array
  • Heap: O(n log k) but maintains streaming capability
  • QuickSelect: O(n^2) worst case without randomization

Red flags: Not knowing QuickSelect exists or its trade-offs.

3. "How would you maintain a running median of a stream?"

What they're looking for:

  • Two heaps: max-heap for lower half, min-heap for upper half
  • Balance sizes (differ by at most 1)
  • Median = top of larger heap or average of both tops

Red flags: Sorting the entire stream for each query.

4. "Why is Merge Sort preferred for external sorting?"

What they're looking for:

  • Sequential disk access (merge reads sequentially from runs)
  • Predictable I/O pattern (no random access like QuickSort)
  • Stable sorting often required for databases

Red flags: Not understanding disk I/O constraints.

5. "How would you sort 1TB of data with only 1GB of RAM?"

What they're looking for:

  • External merge sort: create 1000 sorted runs, k-way merge
  • Optimization: use multiple disk buffers to overlap I/O and CPU
  • Real systems: replacement selection for longer initial runs

Red flags: Only knowing in-memory sorting algorithms.
