Learning Objectives

1. Monotonic Stack (The Most Important Pattern)

Core Idea & Template ▼

A monotonic stack maintains elements in sorted order (either increasing or decreasing) from bottom to top. When a new element violates the order, we pop elements and process them. This lets us find the next greater, next smaller, previous greater, or previous smaller element for every position in O(n) total time.

Why O(n)? Each element is pushed at most once and popped at most once, so across all iterations the total work is 2n = O(n).

Template: Next Greater Element

def next_greater(nums):
    """For each element, find the next element that is strictly greater."""
    n = len(nums)
    result = [-1] * n
    stack = []  # stores indices, monotonic decreasing values

    for i in range(n):
        # Pop everything smaller than nums[i]
        while stack and nums[stack[-1]] < nums[i]:
            idx = stack.pop()
            result[idx] = nums[i]
        stack.append(i)

    return result

# Example: nums = [2, 1, 2, 4, 3]
# result =        [4, 2, 4,-1,-1]

Key insight: The stack stores indices (not values) so we can write answers back to the correct position. The stack's values are monotonically decreasing: when we encounter a larger element, everything smaller gets its answer.

LC 739: Daily Temperatures ▼

Problem: Given daily temperatures, return how many days until a warmer day. This is exactly "next greater element" but returning the distance instead of the value.

def dailyTemperatures(temperatures):
    n = len(temperatures)
    answer = [0] * n
    stack = []  # indices, monotonic decreasing temps

    for i in range(n):
        while stack and temperatures[stack[-1]] < temperatures[i]:
            idx = stack.pop()
            answer[idx] = i - idx  # distance, not value
        stack.append(i)

    return answer
LC 84: Largest Rectangle in Histogram (Hard) ▼

Why this matters: This is one of the most-asked hard stack problems. It combines "previous smaller" and "next smaller" into a single pass. For each bar, we need to know how far left and right it can extend as the shortest bar.

def largestRectangleArea(heights):
    stack = []  # indices, monotonic increasing heights
    max_area = 0
    heights.append(0)  # sentinel to flush remaining bars

    for i, h in enumerate(heights):
        # When we see a shorter bar, pop and compute areas
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            # Width: from current stack top (prev smaller) to i
            width = i if not stack else i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)

    heights.pop()  # clean up sentinel
    return max_area

Key insight: When we pop index j, the current i is the next smaller to the right, and stack[-1] after popping is the previous smaller to the left. The width of the rectangle with height heights[j] is i - stack[-1] - 1.

2. Parenthesis Problems

LC 20: Valid Parentheses ▼

The foundational stack problem. Push opening brackets; on a closing bracket, check the top matches. Two edge cases: stack empty on close (extra closer), stack non-empty at end (extra opener).

def isValid(s):
    stack = []
    match = {')': '(', ']': '[', '}': '{'}

    for ch in s:
        if ch in match:
            if not stack or stack[-1] != match[ch]:
                return False
            stack.pop()
        else:
            stack.append(ch)

    return len(stack) == 0
LC 22: Generate Parentheses & LC 1249: Minimum Remove ▼

Generate Parentheses (Backtracking + Stack invariant)

Generate all valid combinations of n pairs. The key constraint: at any point, open_count ≥ close_count.

def generateParenthesis(n):
    result = []

    def backtrack(current, open_count, close_count):
        if len(current) == 2 * n:
            result.append(current)
            return
        if open_count < n:
            backtrack(current + '(', open_count + 1, close_count)
        if close_count < open_count:
            backtrack(current + ')', open_count, close_count + 1)

    backtrack('', 0, 0)
    return result

Minimum Remove to Make Valid (LC 1249)

Use a stack to track indices of unmatched parentheses, then rebuild the string without them.

def minRemoveToMakeValid(s):
    indices_to_remove = set()
    stack = []  # indices of unmatched '('

    for i, ch in enumerate(s):
        if ch == '(':
            stack.append(i)
        elif ch == ')':
            if stack:
                stack.pop()  # matched
            else:
                indices_to_remove.add(i)  # unmatched ')'

    indices_to_remove.update(stack)  # remaining unmatched '('
    return ''.join(ch for i, ch in enumerate(s) if i not in indices_to_remove)

3. Expression Evaluation

LC 150: Reverse Polish Notation & LC 224: Basic Calculator ▼

Reverse Polish Notation (RPN)

RPN eliminates the need for parentheses and operator precedence. Operands go on the stack; operators pop two operands and push the result.

def evalRPN(tokens):
    stack = []
    ops = {
        '+': lambda a, b: a + b,
        '-': lambda a, b: a - b,
        '*': lambda a, b: a * b,
        '/': lambda a, b: int(a / b),  # truncate toward zero
    }

    for token in tokens:
        if token in ops:
            b, a = stack.pop(), stack.pop()  # note order!
            stack.append(ops[token](a, b))
        else:
            stack.append(int(token))

    return stack[0]

Basic Calculator (LC 224) — handles +, -, (, )

The trick: use a sign stack. When we enter parentheses, push the current sign. When we hit ), pop it. This avoids needing to recursively evaluate sub-expressions.

def calculate(s):
    stack = [1]  # sign stack; 1 = positive context
    result = 0
    num = 0
    sign = 1

    for ch in s:
        if ch.isdigit():
            num = num * 10 + int(ch)
        elif ch in '+-':
            result += sign * num
            num = 0
            sign = stack[-1] * (1 if ch == '+' else -1)
        elif ch == '(':
            stack.append(sign)
        elif ch == ')':
            result += sign * num
            num = 0
            stack.pop()

    return result + sign * num

4. Monotonic Deque: Sliding Window Maximum

LC 239: Sliding Window Maximum ▼

Problem: Given an array and window size k, return the max of each window. Brute force is O(nk). A monotonic deque solves it in O(n).

Why a deque? We need to pop from both ends: from the back (to maintain decreasing order) and from the front (to evict elements outside the window).

from collections import deque

def maxSlidingWindow(nums, k):
    dq = deque()  # stores indices; values are monotonically decreasing
    result = []

    for i, num in enumerate(nums):
        # Remove elements outside the window
        while dq and dq[0] < i - k + 1:
            dq.popleft()

        # Maintain decreasing order: remove smaller elements from back
        while dq and nums[dq[-1]] <= num:
            dq.pop()

        dq.append(i)

        # Window is full; record the max (front of deque)
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

Interview tip: Interviewers often ask "why not a heap?" A max-heap gives O(n log k) because you cannot efficiently remove elements that leave the window. The deque avoids this by eagerly discarding elements that can never be the answer.

5. Design Problems

LC 155: Min Stack & LC 232: Queue Using Two Stacks ▼

Min Stack — O(1) getMin

Keep a parallel stack that tracks the running minimum. Each entry in the min stack corresponds to "what is the minimum if the main stack has this many elements?"

class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []  # parallel; min_stack[i] = min of stack[0..i]

    def push(self, val):
        self.stack.append(val)
        min_val = min(val, self.min_stack[-1] if self.min_stack else val)
        self.min_stack.append(min_val)

    def pop(self):
        self.stack.pop()
        self.min_stack.pop()

    def top(self):
        return self.stack[-1]

    def getMin(self):
        return self.min_stack[-1]

Queue Using Two Stacks — Amortized O(1)

Why this works: Stack A (input) collects pushes. Stack B (output) serves pops. We only transfer A to B (reversing order) when B is empty. Each element is moved at most once, so amortized cost per operation is O(1).

class MyQueue:
    def __init__(self):
        self.stack_in = []   # push here
        self.stack_out = []  # pop from here

    def push(self, x):
        self.stack_in.append(x)

    def pop(self):
        self._transfer()
        return self.stack_out.pop()

    def peek(self):
        self._transfer()
        return self.stack_out[-1]

    def empty(self):
        return not self.stack_in and not self.stack_out

    def _transfer(self):
        if not self.stack_out:
            while self.stack_in:
                self.stack_out.append(self.stack_in.pop())

Real-world usage: This pattern appears in message queues where producers and consumers operate at different rates. Kafka's segment architecture is conceptually similar: writes go to the active segment, reads come from older sealed segments.
