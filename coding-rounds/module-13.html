1. Bit Basics

Bitwise Operators & Binary Representation ▼

Every integer is stored as a sequence of bits. Understanding how operators manipulate these bits at the hardware level is the key to all bit manipulation problems.

Shift Operators

  • Left shift (x << k): Multiplies x by 2k. Fills vacated bits with 0.
  • Right shift (x >> k): Divides x by 2k (floor). In most languages, arithmetic right shift preserves the sign bit for negative numbers; logical right shift (>>> in Java) fills with 0.

Two's Complement

Negative numbers use two's complement: to get -x, invert all bits of x and add 1. For a 32-bit integer, -1 is 1111...1111 (all 32 bits set). This means ~x == -x - 1, a relationship that shows up in many bit tricks.

Key insight: -n == ~n + 1. This means n & (-n) isolates the lowest set bit, because negation flips all bits above the lowest set bit and keeps it.

# Binary representation examples
# Python integers have arbitrary precision, so we mask to 32 bits for display
def show_bits(n, width=8):
    """Display the binary representation of n."""
    if n < 0:
        n = n & ((1 << width) - 1)  # two's complement mask
    return format(n, f'0{width}b')

print(show_bits(13))     # 00001101
print(show_bits(-13, 8)) # 11110011  (two's complement)
print(show_bits(13 << 2)) # 00110100  (13 * 4 = 52)
print(show_bits(13 >> 1)) # 00000110  (13 // 2 = 6)

2. Essential Bit Tricks

The Toolkit Every Engineer Needs ▼

Check if Power of 2

A power of 2 has exactly one bit set: 1, 10, 100, 1000... in binary. Subtracting 1 flips that bit and sets all lower bits: 0111.... So n & (n - 1) is zero if and only if n is a power of 2.

def is_power_of_two(n: int) -> bool:
    return n > 0 and (n & (n - 1)) == 0

# Example: 16 = 10000, 15 = 01111 -> 10000 & 01111 = 0  (True)
# Example: 18 = 10010, 17 = 10001 -> 10010 & 10001 = 10000 (False)

Count Set Bits (Brian Kernighan's Algorithm)

n & (n - 1) clears the lowest set bit. Repeat until n becomes 0, counting iterations. This runs in O(k) where k is the number of set bits, not the total bit width.

def count_set_bits(n: int) -> int:
    count = 0
    while n:
        n &= n - 1   # clear lowest set bit
        count += 1
    return count

# Python built-in: bin(n).count('1') or n.bit_count() (3.10+)

Get / Set / Clear / Toggle the i-th Bit

def get_bit(n, i):
    """Return 1 if bit i is set, else 0."""
    return (n >> i) & 1

def set_bit(n, i):
    """Set bit i to 1."""
    return n | (1 << i)

def clear_bit(n, i):
    """Set bit i to 0."""
    return n & ~(1 << i)

def toggle_bit(n, i):
    """Flip bit i."""
    return n ^ (1 << i)

Isolate the Lowest Set Bit

n & (-n) returns a number with only the lowest set bit of n. This is the basis of Binary Indexed Trees (Fenwick Trees) and appears in many problems.

def lowest_set_bit(n: int) -> int:
    return n & (-n)

# 12 = 1100, -12 = ...0100 -> 1100 & 0100 = 0100 = 4

3. XOR Patterns

XOR Properties & Classic Problems ▼

XOR has three properties that make it the star of bit manipulation interviews:

  • Self-inverse: a ^ a = 0
  • Identity: a ^ 0 = a
  • Commutative & Associative: order does not matter

Together these mean: XOR-ing a list of numbers cancels out every number that appears an even number of times, leaving only the odd-count numbers.

LC #136: Single Number

Every element appears twice except one. XOR all elements; duplicates cancel, leaving the unique number.

def single_number(nums: list[int]) -> int:
    result = 0
    for n in nums:
        result ^= n
    return result
# O(n) time, O(1) space

LC #268: Missing Number

Array of n numbers from 0..n with one missing. XOR all values with all indices 0..n; every number cancels except the missing one.

def missing_number(nums: list[int]) -> int:
    xor = len(nums)
    for i, num in enumerate(nums):
        xor ^= i ^ num
    return xor

LC #260: Single Number III (Two Unique Numbers)

Every element appears twice except two elements a and b. XOR all gives a ^ b. Since a != b, at least one bit differs. Use any differing bit to partition all numbers into two groups; XOR within each group recovers a and b separately.

def single_number_iii(nums: list[int]) -> list[int]:
    xor_all = 0
    for n in nums:
        xor_all ^= n

    # Isolate any bit where a and b differ
    diff_bit = xor_all & (-xor_all)  # lowest set bit

    a, b = 0, 0
    for n in nums:
        if n & diff_bit:
            a ^= n
        else:
            b ^= n
    return [a, b]

Why the partition works: Every paired number has both copies in the same group (they share the same bit value at that position), so they cancel. The two unique numbers land in different groups because they differ at that bit.

4. Bit Masking

Subsets, Hamming Distance & Bit Reversal ▼

Enumerating Subsets with Bitmasks

For a set of n elements, each subset maps to an n-bit number where bit i indicates whether element i is included. This gives a clean O(2n) enumeration. Common in DP on subsets (e.g., TSP, assignment problems).

def subsets(nums: list[int]) -> list[list[int]]:
    n = len(nums)
    result = []
    for mask in range(1 << n):  # 0 to 2^n - 1
        subset = []
        for i in range(n):
            if mask & (1 << i):
                subset.append(nums[i])
        result.append(subset)
    return result

LC #461: Hamming Distance

The Hamming distance between two integers is the number of positions where their bits differ. XOR the two numbers (differing bits become 1), then count set bits.

def hamming_distance(x: int, y: int) -> int:
    xor = x ^ y
    count = 0
    while xor:
        xor &= xor - 1  # Brian Kernighan
        count += 1
    return count

LC #190: Reverse Bits

Reverse all 32 bits of a given unsigned integer. Process bit by bit from LSB to MSB.

def reverse_bits(n: int) -> int:
    result = 0
    for _ in range(32):
        result = (result << 1) | (n & 1)
        n >>= 1
    return result

LC #191: Number of 1 Bits

def hamming_weight(n: int) -> int:
    count = 0
    while n:
        n &= n - 1
        count += 1
    return count

5. Practical Applications

Real-World Bit Manipulation ▼

Bloom Filters

A bloom filter is a bit array of m bits with k hash functions. To add an element, set the k hash-indicated bits to 1. To query, check if all k bits are set. False positives are possible (all bits happen to be set by other elements), but false negatives are impossible. Used in databases (Cassandra, LevelDB) to avoid expensive disk reads for keys that definitely do not exist.

Feature Flags & Permissions

Unix file permissions use 3 bits per user class: rwx = read (4), write (2), execute (1). Checking permissions is a single AND: if (perms & WRITE_BIT). This pattern scales to feature flags in applications where each bit represents a feature toggle.

# Permission flags
READ    = 0b100  # 4
WRITE   = 0b010  # 2
EXECUTE = 0b001  # 1

user_perms = READ | WRITE  # 0b110 = 6

can_read  = bool(user_perms & READ)     # True
can_exec  = bool(user_perms & EXECUTE)  # False

# Grant execute
user_perms |= EXECUTE   # 0b111 = 7
# Revoke write
user_perms &= ~WRITE    # 0b101 = 5
