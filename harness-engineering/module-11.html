Module 11: Streaming & Real-time Patterns

Build agent UX that feels instantaneous — even when tool calls interrupt the stream, clients drop behind, or multiple agents run in parallel.

What You'll Learn

  • Why TTFT (time to first token) is the single most important streaming UX metric
  • When to choose SSE vs WebSocket for different agent communication patterns
  • How to handle the hard problem: tool calls that interrupt the token stream
  • Progressive rendering strategies for partial markdown, code, and diagrams
  • AbortController patterns for clean agent cancellation without orphaned processes
  • Backpressure handling when clients can't keep up with token production
  • Multiplexing multiple agent streams over a single connection

1. Why Streaming Matters for Agent UX

The Psychology of Perceived Latency

Non-streaming agents return a wall of text after a 15–45 second delay. Users click away, assume it's broken, or interrupt it. Streaming changes the interaction model completely: users see progress, they read ahead, they decide earlier whether the output is going in the right direction.

Analogy: Think of a taxi vs. watching your Uber approach on a map. Both take the same time, but the live map eliminates anxiety. Streaming tokens is the live map for your AI response.

Key UX Metrics

MetricDefinitionTargetWhy It Matters
TTFTTime to First Token< 300 msEliminates blank-screen anxiety; most impactful
ITLInter-Token Latency< 30 msSmooth reading experience; feels like typing
TTFCTime to First Chunk (tool result)< 1 sShows the agent is "doing work", not frozen
E2EEnd-to-End LatencyContext-dependentMatters less when TTFT is good
Perceived Latency Formula (Nielsen's model):

perceived_latency ≈ TTFT + (token_count × ITL)

With TTFT=200ms, ITL=20ms, 500 tokens:
perceived = 200ms + (500 × 20ms) = 10.2 s total
but user is reading by token 10 (t = 400ms)

Non-streaming: user stares at spinner for 10.2 s, THEN gets all 500 tokens.

The Tool-Call Interruption Problem

Streaming works perfectly until the model emits a tool call. At that point the token stream stops, the harness must execute the tool (potentially taking seconds), and then resume streaming. From the user's perspective the spinner returns unexpectedly. This is the central engineering challenge of streaming agent UX.

Tokens stream
Tool call emitted
Stream paused
Tool executes
Result injected
Tokens resume

The rest of this module shows exactly how to handle each phase of that timeline.

2. SSE vs WebSocket Patterns

Choosing the Right Transport

Server-Sent Events (SSE)

  • One-way: server → client only
  • Standard HTTP/1.1 or HTTP/2
  • Auto-reconnect built in
  • Works through most proxies/CDNs
  • EventSource API in every browser
  • Text-only (UTF-8)
  • No custom framing needed

WebSockets

  • Bidirectional: full duplex
  • Upgraded HTTP connection
  • Manual reconnect logic required
  • May be blocked by strict proxies
  • Binary or text frames
  • Lower per-message overhead after handshake
  • Client can send messages mid-stream

Decision Matrix for Agent Systems

Use CasePreferred TransportReason
Model token streaming to browserSSEOne-way, trivial proxy compat, EventSource auto-reconnect
User can interrupt / cancel mid-streamWebSocket or HTTP DELETEClient needs to send signal while stream is open
Multi-agent orchestration dashboardWebSocketMultiple bidirectional channels, binary telemetry
Serverless / Cloudflare WorkersSSENo persistent socket support in many edge runtimes
Mobile app with unreliable networkSSEHTTP/2 multiplexing + auto-reconnect handles flaky connections better
Anthropic's choice: The Messages API uses SSE for token streaming. Claude Code uses SSE from the cloud API, then forwards over a local IPC channel to the terminal renderer. The bidirectional control (abort, input) goes through a separate HTTP DELETE or signal channel — not the same SSE stream.

SSE Server — Python (FastAPI)

"""
SSE streaming endpoint for an agent harness.
POST /chat returns a text/event-stream with typed events.
"""
from fastapi import FastAPI, Request
from fastapi.responses import StreamingResponse
import anthropic, asyncio, json

app = FastAPI()
client = anthropic.AsyncAnthropic()

async def _sse_event(event_type: str, data: dict) -> str:
    """Format a single SSE event."""
    payload = json.dumps(data, ensure_ascii=False)
    return f"event: {event_type}\ndata: {payload}\n\n"

async def agent_stream(messages: list, session_id: str):
    """
    Yield SSE-formatted events from an Anthropic streaming call.
    Handles text deltas, tool calls, and stream lifecycle.
    """
    tools = [
        {
            "name": "run_bash",
            "description": "Execute a bash command and return output",
            "input_schema": {
                "type": "object",
                "properties": {
                    "command": {"type": "string"}
                },
                "required": ["command"]
            }
        }
    ]

    yield await _sse_event("stream_start", {"session_id": session_id})

    async with client.messages.stream(
        model="claude-opus-4-5",
        max_tokens=4096,
        messages=messages,
        tools=tools,
    ) as stream:
        async for event in stream:
            if event.type == "content_block_delta":
                if hasattr(event.delta, "text"):
                    # Pure text token — forward immediately
                    yield await _sse_event("text_delta", {
                        "text": event.delta.text,
                        "index": event.index,
                    })
                elif hasattr(event.delta, "partial_json"):
                    # Tool-call argument accumulating
                    yield await _sse_event("tool_input_delta", {
                        "partial_json": event.delta.partial_json,
                        "index": event.index,
                    })

            elif event.type == "content_block_start":
                if event.content_block.type == "tool_use":
                    yield await _sse_event("tool_use_start", {
                        "tool_name": event.content_block.name,
                        "tool_use_id": event.content_block.id,
                        "index": event.index,
                    })

            elif event.type == "content_block_stop":
                yield await _sse_event("content_block_stop", {"index": event.index})

            elif event.type == "message_stop":
                yield await _sse_event("message_stop", {
                    "stop_reason": (await stream.get_final_message()).stop_reason
                })

    yield await _sse_event("stream_end", {"session_id": session_id})


@app.post("/chat/{session_id}")
async def chat_stream(session_id: str, request: Request):
    body = await request.json()
    messages = body.get("messages", [])
    return StreamingResponse(
        agent_stream(messages, session_id),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",   # Disable nginx buffering
            "Connection": "keep-alive",
        }
    )

WebSocket Server — TypeScript (Node.js)

/**
 * WebSocket agent server — bidirectional:
 *   client → server: { type: "message", content: string }
 *                    { type: "abort" }
 *   server → client: { type: "text_delta", text: string }
 *                    { type: "tool_start", toolName: string }
 *                    { type: "tool_result", toolName: string, output: string }
 *                    { type: "done" }
 *                    { type: "error", message: string }
 */
import { WebSocketServer, WebSocket } from "ws";
import Anthropic from "@anthropic-ai/sdk";

const wss = new WebSocketServer({ port: 8080 });
const anthropic = new Anthropic();

interface SessionState {
  abortController: AbortController;
  messages: Anthropic.MessageParam[];
}

const sessions = new Map<WebSocket, SessionState>();

wss.on("connection", (ws: WebSocket) => {
  const state: SessionState = {
    abortController: new AbortController(),
    messages: [],
  };
  sessions.set(ws, state);

  ws.on("message", async (raw: Buffer) => {
    const msg = JSON.parse(raw.toString());

    if (msg.type === "abort") {
      state.abortController.abort();
      state.abortController = new AbortController(); // reset for next turn
      ws.send(JSON.stringify({ type: "aborted" }));
      return;
    }

    if (msg.type === "message") {
      state.messages.push({ role: "user", content: msg.content });
      await streamAgentTurn(ws, state);
    }
  });

  ws.on("close", () => {
    state.abortController.abort();
    sessions.delete(ws);
  });
});

async function streamAgentTurn(
  ws: WebSocket,
  state: SessionState
): Promise<void> {
  try {
    const stream = anthropic.messages.stream(
      {
        model: "claude-opus-4-5",
        max_tokens: 4096,
        messages: state.messages,
      },
      { signal: state.abortController.signal }
    );

    for await (const event of stream) {
      if (ws.readyState !== WebSocket.OPEN) break;

      if (
        event.type === "content_block_delta" &&
        event.delta.type === "text_delta"
      ) {
        ws.send(JSON.stringify({ type: "text_delta", text: event.delta.text }));
      }
    }

    ws.send(JSON.stringify({ type: "done" }));
  } catch (err: any) {
    if (err.name !== "AbortError") {
      ws.send(JSON.stringify({ type: "error", message: err.message }));
    }
  }
}

3. Streaming with Tool Call Interleaving

The Hard Problem of Interrupted Streams

A model turn is not a single token stream — it is a sequence of content blocks. Each block is either a text block or a tool-use block. Both can be partially streamed. The client must accumulate, render, and act on them in order.

Anthropic Streaming Event Taxonomy

Event TypePayloadClient Action
message_startmodel, usage (input tokens)Initialize message state, show usage
content_block_startindex, type ("text" or "tool_use"), id, nameOpen new render slot
content_block_delta (text)delta.textAppend to current text block, re-render markdown
content_block_delta (tool_input)delta.partial_jsonAccumulate JSON string; don't execute yet
content_block_stopindexIf tool_use block: parse JSON, execute tool
message_deltastop_reason, usage (output tokens)Update token counters
message_stop—Finalize render; inject tool results for next turn
Critical: Never execute a tool call while its content_block_delta events are still arriving. The JSON input is streamed in fragments; executing on a partial JSON string will cause a parse error. Wait for content_block_stop to signal the block is complete.

StreamProcessor — Handles Interleaved Events

"""
StreamProcessor: accumulates streaming events, executes tool calls,
and emits a unified render stream to the frontend.
"""
import json, asyncio
from dataclasses import dataclass, field
from typing import AsyncIterator, Callable, Awaitable
import anthropic

@dataclass
class ContentBlock:
    block_type: str          # "text" or "tool_use"
    index: int
    text: str = ""
    tool_id: str = ""
    tool_name: str = ""
    tool_input_raw: str = ""
    tool_input: dict = field(default_factory=dict)


class StreamProcessor:
    """
    Wraps an Anthropic message stream and yields render-ready events,
    executing tool calls inline and continuing the stream.
    """

    def __init__(
        self,
        client: anthropic.AsyncAnthropic,
        tool_executor: Callable[[str, dict], Awaitable[str]],
        model: str = "claude-opus-4-5",
        max_iterations: int = 10,
    ):
        self._client = client
        self._execute_tool = tool_executor
        self._model = model
        self._max_iterations = max_iterations

    async def run(
        self,
        messages: list,
        tools: list,
        on_event: Callable[[str, dict], Awaitable[None]],
    ) -> list:
        """
        Run agentic loop with streaming. Calls on_event for each render event.
        Returns the final messages list (including assistant turns + tool results).
        """
        msgs = list(messages)

        for iteration in range(self._max_iterations):
            blocks: list[ContentBlock] = []
            current_block: ContentBlock | None = None

            await on_event("turn_start", {"iteration": iteration})

            async with self._client.messages.stream(
                model=self._model,
                max_tokens=8192,
                messages=msgs,
                tools=tools,
            ) as stream:
                async for event in stream:

                    if event.type == "content_block_start":
                        cb = event.content_block
                        if cb.type == "text":
                            current_block = ContentBlock("text", event.index)
                        elif cb.type == "tool_use":
                            current_block = ContentBlock(
                                "tool_use", event.index,
                                tool_id=cb.id, tool_name=cb.name
                            )
                            await on_event("tool_use_start", {
                                "index": event.index,
                                "tool_name": cb.name,
                                "tool_use_id": cb.id,
                            })
                        blocks.append(current_block)

                    elif event.type == "content_block_delta":
                        cb = blocks[event.index]
                        if event.delta.type == "text_delta":
                            cb.text += event.delta.text
                            await on_event("text_delta", {
                                "index": event.index,
                                "text": event.delta.text,
                            })
                        elif event.delta.type == "input_json_delta":
                            cb.tool_input_raw += event.delta.partial_json
                            # Show the user the tool input is being formed
                            await on_event("tool_input_delta", {
                                "index": event.index,
                                "partial_json": event.delta.partial_json,
                            })

                    elif event.type == "content_block_stop":
                        cb = blocks[event.index]
                        if cb.block_type == "tool_use":
                            # Safe to parse — block is complete
                            try:
                                cb.tool_input = json.loads(cb.tool_input_raw or "{}")
                            except json.JSONDecodeError as e:
                                await on_event("error", {"message": f"Tool JSON parse error: {e}"})
                                cb.tool_input = {}

                    elif event.type == "message_stop":
                        final = await stream.get_final_message()
                        stop_reason = final.stop_reason
                        usage = final.usage
                        await on_event("message_stop", {
                            "stop_reason": stop_reason,
                            "input_tokens": usage.input_tokens,
                            "output_tokens": usage.output_tokens,
                        })

            # Build assistant message from accumulated blocks
            assistant_content = []
            for blk in blocks:
                if blk.block_type == "text" and blk.text:
                    assistant_content.append({"type": "text", "text": blk.text})
                elif blk.block_type == "tool_use":
                    assistant_content.append({
                        "type": "tool_use",
                        "id": blk.tool_id,
                        "name": blk.tool_name,
                        "input": blk.tool_input,
                    })

            msgs.append({"role": "assistant", "content": assistant_content})

            # Execute all tool calls, collect results
            tool_result_content = []
            for blk in blocks:
                if blk.block_type == "tool_use":
                    await on_event("tool_executing", {
                        "tool_name": blk.tool_name,
                        "tool_use_id": blk.tool_id,
                        "input": blk.tool_input,
                    })
                    try:
                        output = await self._execute_tool(blk.tool_name, blk.tool_input)
                        is_error = False
                    except Exception as exc:
                        output = str(exc)
                        is_error = True

                    await on_event("tool_result", {
                        "tool_name": blk.tool_name,
                        "tool_use_id": blk.tool_id,
                        "output": output,
                        "is_error": is_error,
                    })
                    tool_result_content.append({
                        "type": "tool_result",
                        "tool_use_id": blk.tool_id,
                        "content": output,
                        "is_error": is_error,
                    })

            if not tool_result_content:
                # No tool calls → agent is done
                await on_event("agent_done", {"iterations": iteration + 1})
                break

            msgs.append({"role": "user", "content": tool_result_content})

        return msgs

4. Progressive Rendering

Rendering Partial Output as Tokens Arrive

Raw markdown rendered mid-stream produces ugly flickering — incomplete headings, broken code fences, half-rendered tables. Progressive rendering requires detecting what type of content is being streamed and applying the right rendering strategy per block type.

Progressive Rendering Strategies by Content Type

Content TypeDetectionStrategy
Plain proseNo special prefixRender markdown inline; safe to show partial sentences
Code blockStarts with ```langBuffer until ``` close OR show raw text with code font while incomplete
TableLine starts with |Buffer entire table; render on \n\n or block end
Mermaid diagram```mermaidBuffer until closing fence; call mermaid.render() post-stream
JSON / XMLStarts with { or <Show as monospace text; syntax-highlight on complete
Tool execution progresstool_executing eventShow spinner with tool name + input preview

ProgressiveRenderer (TypeScript/DOM)

/**
 * ProgressiveRenderer: manages the DOM rendering of a streaming agent response.
 * Handles text deltas, code blocks, tables, and tool call display.
 */
export class ProgressiveRenderer {
  private container: HTMLElement;
  private currentTextEl: HTMLElement | null = null;
  private blocks: Map<number, RenderBlock> = new Map();
  private markdownParser: (md: string) => string; // e.g., marked.parse

  constructor(container: HTMLElement, markdownParser: (s: string) => string) {
    this.container = container;
    this.markdownParser = markdownParser;
  }

  onTextDelta(index: number, text: string): void {
    let block = this.blocks.get(index);
    if (!block) {
      block = new TextRenderBlock(this.container);
      this.blocks.set(index, block);
    }
    (block as TextRenderBlock).appendText(text);
  }

  onToolUseStart(index: number, toolName: string, toolUseId: string): void {
    const block = new ToolRenderBlock(this.container, toolName, toolUseId);
    this.blocks.set(index, block);
  }

  onToolInputDelta(index: number, partialJson: string): void {
    const block = this.blocks.get(index) as ToolRenderBlock | undefined;
    block?.appendInputJson(partialJson);
  }

  onToolExecuting(toolName: string, toolUseId: string, input: unknown): void {
    for (const block of this.blocks.values()) {
      if (block instanceof ToolRenderBlock && block.toolUseId === toolUseId) {
        block.showExecuting(input);
      }
    }
  }

  onToolResult(toolUseId: string, output: string, isError: boolean): void {
    for (const block of this.blocks.values()) {
      if (block instanceof ToolRenderBlock && block.toolUseId === toolUseId) {
        block.showResult(output, isError);
      }
    }
  }

  finalizeAll(): void {
    for (const block of this.blocks.values()) {
      block.finalize(this.markdownParser);
    }
    // Trigger Mermaid rendering for any diagram blocks
    if (typeof (window as any).mermaid !== "undefined") {
      (window as any).mermaid.run({ querySelector: ".mermaid" });
    }
  }
}

abstract class RenderBlock {
  abstract finalize(markdownParser: (s: string) => string): void;
}

class TextRenderBlock extends RenderBlock {
  private el: HTMLElement;
  private rawText: string = "";
  private inCodeFence = false;
  private codeFenceBuffer = "";

  constructor(container: HTMLElement) {
    super();
    this.el = document.createElement("div");
    this.el.className = "prose-block";
    container.appendChild(this.el);
  }

  appendText(text: string): void {
    this.rawText += text;
    this._render();
  }

  private _render(): void {
    // Detect incomplete code fence — buffer it to avoid flicker
    const lines = this.rawText.split("\n");
    let safeText = "";
    let pendingFence = "";
    let insideFence = false;
    let fenceStart = -1;

    for (let i = 0; i < lines.length; i++) {
      const line = lines[i];
      if (!insideFence && line.startsWith("```")) {
        insideFence = true;
        fenceStart = i;
        pendingFence = line + "\n";
      } else if (insideFence) {
        pendingFence += line + "\n";
        if (line.trimEnd() === "```") {
          // Complete fence — safe to render
          safeText += pendingFence;
          pendingFence = "";
          insideFence = false;
        }
      } else {
        safeText += line + (i < lines.length - 1 ? "\n" : "");
      }
    }

    if (safeText.trim()) {
      this.el.innerHTML = this._safeParse(safeText);
    }

    if (pendingFence) {
      // Show incomplete code block as preformatted text
      const pre = this.el.querySelector(".pending-code") as HTMLPreElement
        || (() => {
          const p = document.createElement("pre");
          p.className = "pending-code code-block";
          this.el.appendChild(p);
          return p;
        })();
      pre.textContent = pendingFence;
    }
  }

  private _safeParse(md: string): string {
    try {
      return (window as any).marked.parse(md);
    } catch {
      return `<p>${md}</p>`;
    }
  }

  finalize(markdownParser: (s: string) => string): void {
    this.el.innerHTML = markdownParser(this.rawText);
    // Apply Prism.js to all code elements
    this.el.querySelectorAll("pre code").forEach((el) => {
      (window as any).Prism?.highlightElement(el);
    });
  }
}

class ToolRenderBlock extends RenderBlock {
  public toolUseId: string;
  private el: HTMLElement;
  private statusEl: HTMLElement;
  private inputEl: HTMLElement;
  private inputBuffer: string = "";

  constructor(container: HTMLElement, toolName: string, toolUseId: string) {
    super();
    this.toolUseId = toolUseId;
    this.el = document.createElement("div");
    this.el.className = "tool-call-block";
    this.el.innerHTML = `
      <div class="tool-header">
        <span class="tool-icon">⚙</span>
        <strong class="tool-name">${toolName}</strong>
        <span class="tool-status" data-status="building">building input...</span>
      </div>
      <pre class="tool-input-preview code-block"><code></code></pre>
      <div class="tool-result" hidden></div>
    `;
    container.appendChild(this.el);
    this.statusEl = this.el.querySelector(".tool-status")!;
    this.inputEl = this.el.querySelector(".tool-input-preview code")!;
  }

  appendInputJson(partial: string): void {
    this.inputBuffer += partial;
    this.inputEl.textContent = this.inputBuffer;
  }

  showExecuting(input: unknown): void {
    this.statusEl.textContent = "executing...";
    this.statusEl.setAttribute("data-status", "executing");
  }

  showResult(output: string, isError: boolean): void {
    const resultEl = this.el.querySelector(".tool-result") as HTMLElement;
    resultEl.removeAttribute("hidden");
    resultEl.innerHTML = `
      <div class="result-label ${isError ? "error" : "success"}">
        ${isError ? "Error" : "Output"}
      </div>
      <pre class="code-block"><code>${output}</code></pre>
    `;
    this.statusEl.textContent = isError ? "failed" : "completed";
    this.statusEl.setAttribute("data-status", isError ? "error" : "done");
  }

  finalize(_: (s: string) => string): void {
    // Tool blocks are finalized via showResult
  }
}
Mermaid diagrams: Never attempt to render a Mermaid diagram while tokens are still arriving. Buffer the entire ```mermaid ... ``` block, then call mermaid.run() exactly once after stream end. Calling it on partial diagrams produces parse errors that prevent later rendering.

5. Cancellation and Abort

Stopping an Agent Mid-Stream Without Orphaned Processes

Users frequently want to interrupt agents. Maybe the output is wrong, maybe they see the agent is going down a bad path, maybe they just changed their mind. Clean cancellation is deceptively complex in agentic systems because work may be happening at multiple layers simultaneously:

  • An in-flight HTTP call to the LLM API
  • A running subprocess (bash command, compiler, test runner)
  • A pending database transaction
  • An async queue of future tool calls already scheduled

The AbortController Pattern

/**
 * CancellableAgentLoop: wraps StreamProcessor with a clean abort mechanism.
 * Cancelling stops the LLM stream AND any in-flight tool subprocesses.
 */
import { spawn, ChildProcess } from "child_process";
import Anthropic from "@anthropic-ai/sdk";

interface RunningTool {
  toolUseId: string;
  process?: ChildProcess;
  promise: Promise<string>;
  reject: (reason: Error) => void;
}

export class CancellableAgentLoop {
  private abortController = new AbortController();
  private runningTools = new Map<string, RunningTool>();
  private client: Anthropic;

  constructor(apiKey: string) {
    this.client = new Anthropic({ apiKey });
  }

  /** Call this to stop everything. Safe to call multiple times. */
  abort(): void {
    if (this.abortController.signal.aborted) return;
    this.abortController.abort();
    this._killAllTools();
  }

  /** Reset for a new conversation turn. */
  reset(): void {
    this.abortController = new AbortController();
  }

  private _killAllTools(): void {
    for (const [id, tool] of this.runningTools) {
      tool.process?.kill("SIGTERM");
      // Give it 500ms then SIGKILL
      setTimeout(() => tool.process?.kill("SIGKILL"), 500);
      tool.reject(new Error("Aborted by user"));
    }
    this.runningTools.clear();
  }

  async executeTool(toolName: string, input: Record<string, unknown>, toolUseId: string): Promise<string> {
    if (this.abortController.signal.aborted) {
      throw new Error("Agent is aborted");
    }

    if (toolName === "run_bash") {
      return this._runBash(input.command as string, toolUseId);
    }
    // Add other tools here
    throw new Error(`Unknown tool: ${toolName}`);
  }

  private _runBash(command: string, toolUseId: string): Promise<string> {
    let rejectFn!: (reason: Error) => void;

    const promise = new Promise<string>((resolve, reject) => {
      rejectFn = reject;

      const child = spawn("bash", ["-c", command], {
        timeout: 30_000,
        env: {
          ...process.env,
          // Restrict environment for safety
          PATH: "/usr/local/bin:/usr/bin:/bin",
        },
      });

      const runningTool: RunningTool = {
        toolUseId,
        process: child,
        promise,
        reject,
      };
      this.runningTools.set(toolUseId, runningTool);

      let stdout = "";
      let stderr = "";

      child.stdout.on("data", (d) => (stdout += d.toString()));
      child.stderr.on("data", (d) => (stderr += d.toString()));

      child.on("close", (code) => {
        this.runningTools.delete(toolUseId);
        if (code === 0) {
          resolve(stdout || "(no output)");
        } else {
          resolve(`Exit ${code}\n${stderr || stdout}`);
        }
      });

      child.on("error", (err) => {
        this.runningTools.delete(toolUseId);
        reject(err);
      });
    });

    return promise;
  }

  async run(
    messages: Anthropic.MessageParam[],
    tools: Anthropic.Tool[],
    onEvent: (type: string, data: unknown) => void
  ): Promise<void> {
    const signal = this.abortController.signal;

    try {
      const stream = this.client.messages.stream(
        {
          model: "claude-opus-4-5",
          max_tokens: 8192,
          messages,
          tools,
        },
        { signal }
      );

      for await (const event of stream) {
        if (signal.aborted) {
          onEvent("aborted", {});
          return;
        }
        onEvent(event.type, event);
      }
    } catch (err: any) {
      if (err.name === "AbortError" || signal.aborted) {
        onEvent("aborted", {});
      } else {
        onEvent("error", { message: err.message });
      }
    } finally {
      this._killAllTools();
    }
  }
}
Orphaned processes: If your tool executor spawns child processes, make sure abort() reaches them. A common bug is aborting the HTTP stream but leaving a bash subprocess still running a database migration or a long test suite. Always store a reference to the child process and SIGTERM/SIGKILL it explicitly.

6. Backpressure in Streaming

When Clients Can't Keep Up

Modern LLMs can produce tokens faster than slow client connections can consume them. When the server-side buffer fills up, you have a backpressure problem. Unhandled backpressure causes connection drops, missed tokens, or unbounded memory usage on the server.

Token Production vs Consumption:

Claude Opus: ~50-80 tokens/sec
Average token size: ~4 bytes
Token production rate: ~200-320 bytes/sec

Mobile 3G uplink: 384 kbps ÷ 8 = 48 KB/sec

Conclusion: 3G is fast enough for token streaming.
But on a 2G connection (14.4 kbps = 1.8 KB/sec),
a server buffering aggressively can accumulate
tokens faster than they drain, growing the buffer
unboundedly over a long response.

Backpressure Strategies

StrategyMechanismTrade-off

BackpressureAwareStream — Python

"""
BackpressureAwareStream: wraps an async generator with a bounded
asyncio.Queue to enforce flow control between the Anthropic API
consumer and the SSE sender.
"""
import asyncio
from typing import AsyncIterator, Optional
import anthropic

_SENTINEL = object()  # Signals end of stream

class BackpressureAwareStream:
    """
    Decouples the fast Anthropic stream reader from the potentially
    slow HTTP response writer via a bounded async queue.
    """

    def __init__(self, max_buffer: int = 256):
        self._queue: asyncio.Queue = asyncio.Queue(maxsize=max_buffer)
        self._producer_task: Optional[asyncio.Task] = None
        self._closed = False

    async def start(
        self,
        client: anthropic.AsyncAnthropic,
        messages: list,
        model: str = "claude-opus-4-5",
        max_tokens: int = 4096,
    ) -> None:
        """Start the background producer task."""
        self._producer_task = asyncio.create_task(
            self._produce(client, messages, model, max_tokens)
        )

    async def _produce(self, client, messages, model, max_tokens):
        """
        Reads from Anthropic stream as fast as possible, but blocks
        when the queue is full — naturally applying backpressure back
        to the SDK's async iterator.
        """
        try:
            async with client.messages.stream(
                model=model,
                max_tokens=max_tokens,
                messages=messages,
            ) as stream:
                async for event in stream:
                    # This put() will block if queue is full —
                    # that's the backpressure mechanism!
                    await self._queue.put(event)
        except asyncio.CancelledError:
            pass
        except Exception as exc:
            await self._queue.put(exc)  # Propagate error to consumer
        finally:
            await self._queue.put(_SENTINEL)

    async def events(self) -> AsyncIterator:
        """Consume events. Will pause naturally when queue is empty."""
        while True:
            item = await self._queue.get()
            if item is _SENTINEL:
                break
            if isinstance(item, Exception):
                raise item
            yield item
            self._queue.task_done()

    async def close(self):
        """Cancel the producer and drain the queue."""
        if self._producer_task:
            self._producer_task.cancel()
            try:
                await self._producer_task
            except asyncio.CancelledError:
                pass
        self._closed = True

    @property
    def buffer_fill_ratio(self) -> float:
        """Returns 0.0 (empty) to 1.0 (full). Use for monitoring."""
        if self._queue.maxsize == 0:
            return 0.0
        return self._queue.qsize() / self._queue.maxsize

    @property
    def is_healthy(self) -> bool:
        """Stream is healthy if buffer is not more than 80% full."""
        return self.buffer_fill_ratio < 0.8


# ── Usage in FastAPI endpoint ──────────────────────────────────────────
async def sse_with_backpressure(messages: list, client):
    stream = BackpressureAwareStream(max_buffer=128)
    await stream.start(client, messages)

    try:
        async for event in stream.events():
            # Log buffer health for observability
            if not stream.is_healthy:
                # Optionally pause or alert
                pass

            if event.type == "content_block_delta" and hasattr(event.delta, "text"):
                payload = json.dumps({"text": event.delta.text})
                yield f"event: text_delta\ndata: {payload}\n\n"

            elif event.type == "message_stop":
                yield f"event: done\ndata: {{}}\n\n"
    finally:
        await stream.close()
Python async generators ARE backpressure: If your SSE generator is a Python async def with yield, and the FastAPI/Starlette response writer is awaiting each chunk, then when the network is slow, the writer awaits longer, the generator pauses at the next yield, and the Anthropic SDK's async for pauses at its next iteration. Backpressure propagates automatically through the async chain — no queue needed for the simple case. The queue pattern above is needed when you have a fanout (one stream, multiple consumers) or when the producer and consumer run in truly separate tasks.

7. Multi-Stream Coordination

Multiple Agents Streaming Simultaneously

Multi-agent systems (like a planner + executor + verifier running in parallel) produce multiple concurrent streams. The frontend needs to display all of them clearly and efficiently. Opening a separate SSE connection per agent works but is wasteful. A multiplexed stream over a single WebSocket is far more efficient.

StreamMultiplexer — TypeScript

/**
 * StreamMultiplexer: fans in multiple agent streams and forwards them
 * over a single WebSocket connection, tagged by agent ID.
 *
 * Protocol (server → client):
 * { agent_id: string, seq: number, type: string, data: unknown }
 *
 * Protocol (client → server):
 * { type: "abort", agent_id: string }
 * { type: "pause", agent_id: string }
 * { type: "resume", agent_id: string }
 */
import { WebSocket } from "ws";
import Anthropic from "@anthropic-ai/sdk";

interface AgentStream {
  agentId: string;
  label: string;
  abortController: AbortController;
  paused: boolean;
  seq: number;
}

export class StreamMultiplexer {
  private ws: WebSocket;
  private client: Anthropic;
  private agents = new Map<string, AgentStream>();
  private pauseQueues = new Map<string, unknown[]>();

  constructor(ws: WebSocket, client: Anthropic) {
    this.ws = ws;
    this.client = client;

    ws.on("message", (raw: Buffer) => {
      const msg = JSON.parse(raw.toString());
      this._handleClientMessage(msg);
    });
  }

  async spawnAgent(
    agentId: string,
    label: string,
    messages: Anthropic.MessageParam[]
  ): Promise<void> {
    const agentStream: AgentStream = {
      agentId,
      label,
      abortController: new AbortController(),
      paused: false,
      seq: 0,
    };
    this.agents.set(agentId, agentStream);
    this.pauseQueues.set(agentId, []);

    this._send(agentId, "agent_start", { label });

    try {
      const stream = this.client.messages.stream(
        {
          model: "claude-opus-4-5",
          max_tokens: 4096,
          messages,
        },
        { signal: agentStream.abortController.signal }
      );

      for await (const event of stream) {
        if (agentStream.paused) {
          // Buffer events while paused
          this.pauseQueues.get(agentId)!.push(event);
          continue;
        }
        this._forwardEvent(agentId, event);
      }

      this._send(agentId, "agent_done", {});
    } catch (err: any) {
      if (err.name !== "AbortError") {
        this._send(agentId, "agent_error", { message: err.message });
      } else {
        this._send(agentId, "agent_aborted", {});
      }
    } finally {
      this.agents.delete(agentId);
      this.pauseQueues.delete(agentId);
    }
  }

  private _forwardEvent(agentId: string, event: unknown): void {
    const agent = this.agents.get(agentId);
    if (!agent) return;
    this._send(agentId, (event as any).type, event);
  }

  private _send(agentId: string, type: string, data: unknown): void {
    const agent = this.agents.get(agentId);
    const seq = agent ? agent.seq++ : 0;

    if (this.ws.readyState !== WebSocket.OPEN) return;
    this.ws.send(
      JSON.stringify({ agent_id: agentId, seq, type, data })
    );
  }

  private _handleClientMessage(msg: { type: string; agent_id: string }): void {
    const agent = this.agents.get(msg.agent_id);
    if (!agent) return;

    if (msg.type === "abort") {
      agent.abortController.abort();
    } else if (msg.type === "pause") {
      agent.paused = true;
    } else if (msg.type === "resume") {
      agent.paused = false;
      // Drain buffered events
      const queue = this.pauseQueues.get(msg.agent_id) ?? [];
      for (const event of queue) {
        this._forwardEvent(msg.agent_id, event);
      }
      this.pauseQueues.set(msg.agent_id, []);
    }
  }

  /** Priority ordering: give high-priority agents more CPU budget */
  setPriority(_agentId: string, _priority: "high" | "normal" | "low"): void {
    // In Node.js, we can use setImmediate vs setTimeout(0) vs setTimeout(100)
    // for different agent event processing priorities. Implementation omitted
    // for brevity — the key concept is interleaving event processing.
  }
}

Client-Side Demultiplexer (TypeScript)

/**
 * Client-side demultiplexer: routes multiplexed WS messages to
 * per-agent ProgressiveRenderer instances.
 */
export class StreamDemultiplexer {
  private ws: WebSocket;
  private renderers = new Map<string, ProgressiveRenderer>();

  constructor(wsUrl: string) {
    this.ws = new WebSocket(wsUrl);
    this.ws.onmessage = (evt) => {
      const msg = JSON.parse(evt.data);
      this._route(msg);
    };
  }

  registerAgent(agentId: string, container: HTMLElement, md: (s: string) => string): void {
    this.renderers.set(agentId, new ProgressiveRenderer(container, md));
  }

  private _route(msg: { agent_id: string; type: string; data: any }): void {
    const renderer = this.renderers.get(msg.agent_id);
    if (!renderer) return;

    switch (msg.type) {
      case "content_block_delta":
        if (msg.data?.delta?.type === "text_delta") {
          renderer.onTextDelta(msg.data.index, msg.data.delta.text);
        }
        break;
      case "content_block_start":
        if (msg.data?.content_block?.type === "tool_use") {
          renderer.onToolUseStart(
            msg.data.index,
            msg.data.content_block.name,
            msg.data.content_block.id
          );
        }
        break;
      case "agent_done":
        renderer.finalizeAll();
        break;
    }
  }

  abortAgent(agentId: string): void {
    this.ws.send(JSON.stringify({ type: "abort", agent_id: agentId }));
  }

  pauseAgent(agentId: string): void {
    this.ws.send(JSON.stringify({ type: "pause", agent_id: agentId }));
  }

  resumeAgent(agentId: string): void {
    this.ws.send(JSON.stringify({ type: "resume", agent_id: agentId }));
  }
}

9. Knowledge Check

Q1. What is TTFT and why is it the most important streaming UX metric?

TTFT = Time to First Token. It is the delay between the user submitting a message and the first visible character appearing on screen. It is the most important metric because it eliminates the "blank screen" anxiety period. Once tokens are flowing, users read along and the remaining wait feels productive rather than unresponsive. A 15-second E2E latency with a 200ms TTFT feels far better than a 5-second E2E with no streaming at all.

Q2. Why is it dangerous to execute a tool call as soon as you see a content_block_start event with type "tool_use"?

The tool's input JSON is streamed in fragments via subsequent content_block_delta events with delta.type = "input_json_delta". At the moment of content_block_start the input is empty or incomplete. Attempting to parse it produces a JSON parse error. The tool must be executed only after the corresponding content_block_stop event, which signals the JSON is fully accumulated.

Q3. You have a streaming agent and the user clicks "Stop". List three layers where cleanup must happen.

  1. LLM API call: Call abortController.abort(); the SDK will close the HTTP connection and stop token generation.
  2. In-flight tool processes: Any ChildProcess (bash, python, etc.) spawned by tool executors must be SIGTERM'd (then SIGKILL after timeout) to prevent orphaned processes consuming CPU.
  3. Queued/pending work: Any tool calls that were scheduled but not yet started (e.g., in an async queue) must be discarded and their promises rejected.
Bonus: Reset the AbortController to a fresh instance so the next turn can be cleanly started.

Q4. When should you choose WebSocket over SSE for an agent system?

Choose WebSocket when:
  • You need the client to send messages while the stream is active (e.g., abort signals, user clarifications)
  • You want to multiplex multiple agent streams over a single connection
  • You need binary data or lower per-message overhead for high-frequency telemetry
  • You need pause/resume control signals
SSE is sufficient (and simpler) when the stream is purely server-to-client and cancellation can be handled via a separate HTTP DELETE or signal endpoint.

Q5. Explain how Python async generators naturally solve backpressure without an explicit queue.

Python async for / yield generators are cooperative: a generator resumes only when the consumer awaits the next value. If the HTTP response writer is slow (awaiting a network write), it does not call __anext__() on the upstream generator, which therefore does not call __anext__() on the Anthropic SDK's async iterator, which in turn does not read the next bytes from the TCP socket. The TCP receive window fills up, the OS signals the server to stop sending, and the Anthropic API's TCP sender backs off. Backpressure propagates all the way from the slow client's network to the LLM API without any explicit buffer or queue.

Checkpoint Summary — Module 11

tl;dr: Streaming agent UX is not just "forward tokens to the browser." It requires careful handling of tool-call interruption, progressive rendering, clean cancellation, backpressure, and (for multi-agent systems) stream multiplexing.

  • TTFT is king. Streaming reduces perceived latency from E2E to TTFT. Target <300ms.
  • SSE for output, WebSocket for control. SSE is simpler and more proxy-compatible. Add a separate cancel endpoint or use WebSocket when the client must send signals mid-stream.
  • Never execute tool calls on partial JSON. Wait for content_block_stop. Buffer input_json_delta fragments into a string before parsing.
  • Progressive rendering needs fence detection. Buffer incomplete code fences and Mermaid blocks. Render prose incrementally. Call Prism.highlightElement() and mermaid.run() post-stream.
  • Cancellation is layered. Abort the HTTP call. Kill child processes. Drain async queues. Reset the AbortController for the next turn.
  • Backpressure is implicit in async generators but needs an explicit bounded queue when producer and consumer are in separate tasks.
  • Multi-agent multiplexing reduces connection count and enables per-agent pause/resume/abort over a single WebSocket.
