Design an Amazon Locker System

Difficulty: Medium-Hard LLD Problem | Time: 40-50 minutes

Design a package locker system (like Amazon Hub Locker) where delivery drivers deposit packages into lockers and customers pick them up using a one-time code. This problem tests state machine design, strategy pattern application, notification systems, and expiry/timeout handling -- all critical for staff-level interviews.

1. Requirements Clarification

Functional Requirements ▼
  • A LockerLocation contains multiple lockers of varying sizes (Small, Medium, Large, Extra-Large)
  • The system assigns a locker to a package based on package dimensions
  • A delivery driver deposits a package into the assigned locker
  • The system generates a unique OTP (one-time password) and sends it to the customer via SMS/email
  • The customer enters the OTP at the locker kiosk to open the locker and retrieve the package
  • Packages that are not picked up within a configurable time window (e.g., 3 days) expire and are returned to sender
  • The system tracks all package state transitions (reserved, deposited, picked up, expired, returned)

Non-Functional Requirements

  • Concurrency: Multiple drivers may deposit at the same location simultaneously
  • Reliability: OTPs must be unique and time-bound; no two active packages share an OTP at the same location
  • Availability: Locker kiosks should work even with intermittent network connectivity
  • Scalability: Support thousands of locker locations across a metro area

Clarifying Questions to Ask

  • "What locker sizes do we support?" (Small, Medium, Large, Extra-Large)
  • "How long before an uncollected package expires?" (Configurable, default 3 days)
  • "Can a customer have multiple packages at the same location?" (Yes, each gets its own locker and OTP)
  • "How is the OTP delivered?" (SMS and email notification)
  • "What happens on expiry?" (Package is flagged for return-to-sender pickup by driver)
  • "Do we need to handle locker hardware/sensors?" (Out of scope for LLD, assume an API contract)
Assumptions ▼
  • Each package has a pre-determined size category (computed from dimensions before delivery)
  • A locker can hold exactly one package at a time
  • OTPs are 6-digit numeric codes, unique per active package at a given location
  • The system runs a scheduled job to detect and handle expired packages
  • Hardware interaction (opening/closing locker door) is abstracted behind an interface

4. Package Lifecycle State Machine

State Transitions ▼

The package lifecycle is the heart of this design. Every package follows a strict state machine -- understanding these transitions and what triggers them is critical in the interview.

stateDiagram-v2 [*] --> RESERVED : System assigns locker RESERVED --> DEPOSITED : Driver deposits package RESERVED --> CANCELLED : Locker reservation timeout DEPOSITED --> PICKED_UP : Customer enters valid OTP DEPOSITED --> EXPIRED : Expiry window exceeded (3 days) EXPIRED --> RETURNED : Driver picks up for return PICKED_UP --> [*] RETURNED --> [*] CANCELLED --> [*]

State Transition Rules

From To Trigger Side Effects
-- RESERVED Locker allocated for incoming package Locker status set to RESERVED
RESERVED DEPOSITED Driver scans package and places in locker OTP generated, notification sent, expiry timer starts
DEPOSITED PICKED_UP Customer enters correct OTP Locker opened, OTP invalidated, locker released
DEPOSITED EXPIRED Scheduled job detects past-due package OTP invalidated, return-to-sender flagged
EXPIRED RETURNED Driver retrieves package for return Locker released back to AVAILABLE pool

5. Design Patterns Used

State Pattern

Where: Package and Locker status management

Why: The package lifecycle is a well-defined state machine. Using the State pattern ensures that only valid transitions are allowed and each state encapsulates its own behavior (e.g., what happens when a DEPOSITED package expires vs. gets picked up).

Strategy Pattern

Where: LockerAllocationStrategy (BestFitStrategy, FirstFitStrategy)

Why: Different allocation algorithms can be swapped at runtime. Best-fit minimizes wasted space (assigns smallest locker that fits). First-fit optimizes for speed. The system can A/B test strategies without code changes.

Observer Pattern

Where: NotificationService with SMS/Email observers

Why: When a package is deposited or nearing expiry, multiple notification channels need to fire. The Observer pattern decouples the delivery logic from notification delivery -- adding a push notification channel requires zero changes to the core system.

Factory Pattern

Where: Locker creation based on configuration

Why: Different locker locations have different size distributions. A factory encapsulates the creation logic, making it easy to configure locations from a spec (e.g., "20 small, 15 medium, 10 large, 5 extra-large").

public class Package {
    private final String packageId;
    private final LockerSize requiredSize;
    private final String customerId;
    private PackageStatus status;

    // Valid state transitions
    private static final Map<PackageStatus, Set<PackageStatus>> VALID_TRANSITIONS = Map.of(
        PackageStatus.CREATED,    Set.of(PackageStatus.IN_TRANSIT),
        PackageStatus.IN_TRANSIT, Set.of(PackageStatus.DEPOSITED),
        PackageStatus.DEPOSITED,  Set.of(PackageStatus.PICKED_UP, PackageStatus.EXPIRED),
        PackageStatus.EXPIRED,    Set.of(PackageStatus.RETURNED),
        PackageStatus.PICKED_UP,  Set.of(),
        PackageStatus.RETURNED,   Set.of()
    );

    public Package(String packageId, LockerSize requiredSize, String customerId) {
        this.packageId = packageId;
        this.requiredSize = requiredSize;
        this.customerId = customerId;
        this.status = PackageStatus.CREATED;
    }

    /**
     * Enforces the state machine -- only valid transitions are permitted.
     *
     * @throws IllegalStateException if the transition is invalid
     */
    public synchronized void transitionTo(PackageStatus newStatus) {
        Set<PackageStatus> allowed = VALID_TRANSITIONS.getOrDefault(
            this.status, Set.of());
        if (!allowed.contains(newStatus)) {
            throw new IllegalStateException(
                "Invalid transition: " + this.status + " -> " + newStatus);
        }
        this.status = newStatus;
    }

    public String getPackageId() { return packageId; }
    public LockerSize getRequiredSize() { return requiredSize; }
    public String getCustomerId() { return customerId; }
    public PackageStatus getStatus() { return status; }

    public PackageInfo toInfo() {
        return new PackageInfo(packageId, requiredSize, customerId, status);
    }
}
Locker - Thread-Safe State Management ▼ 
import java.util.concurrent.locks.ReentrantLock;

public class Locker {
    private final String lockerId;
    private final LockerSize size;
    private LockerStatus status;
    private Package currentPackage;
    private final ReentrantLock lock = new ReentrantLock();

    public Locker(String lockerId, LockerSize size) {
        this.lockerId = lockerId;
        this.size = size;
        this.status = LockerStatus.AVAILABLE;
    }

    public String getLockerId() { return lockerId; }
    public LockerSize getSize() { return size; }

    public LockerStatus getStatus() {
        lock.lock();
        try {
            return status;
        } finally {
            lock.unlock();
        }
    }

    public boolean isAvailable() {
        lock.lock();
        try {
            return status == LockerStatus.AVAILABLE;
        } finally {
            lock.unlock();
        }
    }

    /**
     * Reserves this locker for an incoming package.
     * Thread-safe: uses lock to prevent double-reservation.
     *
     * @return true if reservation succeeded, false if locker was already taken
     */
    public boolean reserve() {
        lock.lock();
        try {
            if (status != LockerStatus.AVAILABLE) {
                return false;
            }
            status = LockerStatus.RESERVED;
            return true;
        } finally {
            lock.unlock();
        }
    }

    /**
     * Deposits a package into this locker.
     *
     * @return true if deposit succeeded
     * @throws IllegalStateException if locker is not in RESERVED state
     */
    public boolean assignPackage(Package pkg) {
        lock.lock();
        try {
            if (status != LockerStatus.RESERVED) {
                throw new IllegalStateException(
                    "Locker " + lockerId + " is not reserved (status: " + status + ")");
            }
            if (!size.canFit(pkg.getRequiredSize())) {
                return false;
            }
            this.currentPackage = pkg;
            this.status = LockerStatus.OCCUPIED;
            return true;
        } finally {
            lock.unlock();
        }
    }

    /**
     * Releases the package and makes the locker available again.
     *
     * @return the released package, or null if locker was empty
     */
    public Package releasePackage() {
        lock.lock();
        try {
            Package pkg = this.currentPackage;
            this.currentPackage = null;
            this.status = LockerStatus.AVAILABLE;
            return pkg;
        } finally {
            lock.unlock();
        }
    }

    /**
     * Cancels a reservation (e.g., reservation timeout).
     */
    public void cancelReservation() {
        lock.lock();
        try {
            if (status == LockerStatus.RESERVED) {
                status = LockerStatus.AVAILABLE;
            }
        } finally {
            lock.unlock();
        }
    }

    public Package getCurrentPackage() {
        lock.lock();
        try {
            return currentPackage;
        } finally {
            lock.unlock();
        }
    }

    @Override
    public String toString() {
        return String.format("Locker[%s, %s, %s]", lockerId, size, status);
    }
}
LockerLocation - Managing Lockers at a Site ▼ 
import java.util.*;
import java.util.concurrent.ConcurrentHashMap;

public class LockerLocation {
    private final String locationId;
    private final String address;
    private final Map<LockerSize, List<Locker>> lockersBySize;
    private final Map<String, Delivery> activeOTPs; // OTP -> Delivery

    public LockerLocation(String locationId, String address) {
        this.locationId = locationId;
        this.address = address;
        this.lockersBySize = new ConcurrentHashMap<>();
        this.activeOTPs = new ConcurrentHashMap<>();

        for (LockerSize size : LockerSize.values()) {
            lockersBySize.put(size, Collections.synchronizedList(new ArrayList<>()));
        }
    }

    public void addLocker(Locker locker) {
        lockersBySize.get(locker.getSize()).add(locker);
    }

    /**
     * Returns all available lockers that can fit the given size.
     * Searches from exact size upward (e.g., MEDIUM package can go
     * into MEDIUM, LARGE, or EXTRA_LARGE locker).
     */
    public List<Locker> getAvailableLockers(LockerSize requiredSize) {
        List<Locker> available = new ArrayList<>();
        for (LockerSize size : LockerSize.values()) {
            if (size.canFit(requiredSize)) {
                lockersBySize.get(size).stream()
                    .filter(Locker::isAvailable)
                    .forEach(available::add);
            }
        }
        return available;
    }

    public void registerOTP(String otp, Delivery delivery) {
        activeOTPs.put(otp, delivery);
    }

    public void invalidateOTP(String otp) {
        activeOTPs.remove(otp);
    }

    public Delivery findDeliveryByOTP(String otp) {
        return activeOTPs.get(otp);
    }

    public int getTotalAvailable() {
        return lockersBySize.values().stream()
            .mapToInt(lockers -> (int) lockers.stream()
                .filter(Locker::isAvailable).count())
            .sum();
    }

    public LocationAvailability getAvailability() {
        Map<LockerSize, Integer> available = new EnumMap<>(LockerSize.class);
        for (LockerSize size : LockerSize.values()) {
            available.put(size, (int) lockersBySize.get(size).stream()
                .filter(Locker::isAvailable).count());
        }
        return new LocationAvailability(locationId, address, available);
    }

    public String getLocationId() { return locationId; }
    public String getAddress() { return address; }
}
LockerAllocationStrategy - Best-Fit and First-Fit ▼ 
import java.util.Comparator;
import java.util.List;
import java.util.Optional;

/**
 * Strategy interface for locker allocation.
 * Different strategies optimize for different goals:
 * - BestFit: minimize wasted space (use smallest locker that fits)
 * - FirstFit: minimize search time (use first available locker that fits)
 */
public interface LockerAllocationStrategy {
    Optional<Locker> allocate(List<Locker> availableLockers, LockerSize requiredSize);
}

/**
 * Best-Fit: Allocates the smallest available locker that fits the package.
 *
 * Why use this? Maximizes locker utilization. A small package won't
 * block a large locker that a bigger package needs later.
 *
 * Trade-off: Slightly slower (must sort/compare all candidates).
 */
public class BestFitStrategy implements LockerAllocationStrategy {
    @Override
    public Optional<Locker> allocate(List<Locker> availableLockers, LockerSize requiredSize) {
        return availableLockers.stream()
            .filter(locker -> locker.getSize().canFit(requiredSize))
            .min(Comparator.comparingInt(locker -> locker.getSize().getCapacity()));
    }
}

/**
 * First-Fit: Allocates the first available locker that fits.
 *
 * Why use this? Fastest allocation time O(1) in best case.
 * Useful when locker utilization is low and speed matters.
 *
 * Trade-off: May waste large lockers on small packages.
 */
public class FirstFitStrategy implements LockerAllocationStrategy {
    @Override
    public Optional<Locker> allocate(List<Locker> availableLockers, LockerSize requiredSize) {
        return availableLockers.stream()
            .filter(locker -> locker.getSize().canFit(requiredSize))
            .findFirst();
    }
}
OTPService - Generating and Validating One-Time Codes ▼ 
import java.security.SecureRandom;
import java.util.Map;
import java.util.Set;
import java.util.concurrent.ConcurrentHashMap;

/**
 * Generates and manages one-time pickup codes.
 *
 * Key design decisions:
 * 1. OTPs are unique per location (not globally) -- reduces collision space
 * 2. Uses SecureRandom, not Random -- prevents OTP prediction attacks
 * 3. 6-digit codes balance usability (easy to type) with security (1M combinations)
 * 4. Thread-safe via ConcurrentHashMap
 */
public class OTPService {
    private static final int OTP_LENGTH = 6;
    private final SecureRandom secureRandom = new SecureRandom();

    // locationId -> set of active OTPs at that location
    private final Map<String, Set<String>> activeOTPsByLocation = new ConcurrentHashMap<>();

    /**
     * Generates a unique 6-digit OTP for the given location.
     * Retries if there is a collision with an existing active OTP.
     */
    public String generate(String locationId) {
        Set<String> activeOTPs = activeOTPsByLocation.computeIfAbsent(
            locationId, k -> ConcurrentHashMap.newKeySet());

        String otp;
        int maxRetries = 100;
        int attempt = 0;
        do {
            otp = String.format("%0" + OTP_LENGTH + "d",
                secureRandom.nextInt((int) Math.pow(10, OTP_LENGTH)));
            attempt++;
            if (attempt > maxRetries) {
                throw new IllegalStateException(
                    "Cannot generate unique OTP after " + maxRetries + " attempts");
            }
        } while (activeOTPs.contains(otp));

        activeOTPs.add(otp);
        return otp;
    }

    /**
     * Validates that the given OTP is active at the specified location.
     */
    public boolean validate(String locationId, String otp) {
        Set<String> activeOTPs = activeOTPsByLocation.get(locationId);
        return activeOTPs != null && activeOTPs.contains(otp);
    }

    /**
     * Invalidates an OTP after successful pickup or expiry.
     */
    public void invalidate(String locationId, String otp) {
        Set<String> activeOTPs = activeOTPsByLocation.get(locationId);
        if (activeOTPs != null) {
            activeOTPs.remove(otp);
        }
    }
}
NotificationService - Observer Pattern ▼ 
import java.util.List;
import java.util.concurrent.CopyOnWriteArrayList;

/**
 * Observer interface for notification channels.
 * Adding a new channel (e.g., push notifications) requires
 * only implementing this interface -- zero core logic changes.
 */
public interface NotificationObserver {
    void onPackageDeposited(PickupNotification notification);
    void onExpiryReminder(PickupNotification notification);
}

/**
 * SMS notification channel.
 */
public class SMSNotificationObserver implements NotificationObserver {
    @Override
    public void onPackageDeposited(PickupNotification notification) {
        System.out.printf("[SMS -> %s] Your package %s is ready for pickup " +
            "at %s. OTP: %s. Expires: %s%n",
            notification.customerId(), notification.packageId(),
            notification.locationAddress(), notification.otp(),
            notification.expiryTime());
    }

    @Override
    public void onExpiryReminder(PickupNotification notification) {
        System.out.printf("[SMS -> %s] Reminder: Package %s expires at %s. " +
            "Pick up soon!%n",
            notification.customerId(), notification.packageId(),
            notification.expiryTime());
    }
}

/**
 * Email notification channel.
 */
public class EmailNotificationObserver implements NotificationObserver {
    @Override
    public void onPackageDeposited(PickupNotification notification) {
        System.out.printf("[EMAIL -> %s] Package %s deposited at %s. " +
            "Use code %s to pick up before %s.%n",
            notification.customerId(), notification.packageId(),
            notification.locationAddress(), notification.otp(),
            notification.expiryTime());
    }

    @Override
    public void onExpiryReminder(PickupNotification notification) {
        System.out.printf("[EMAIL -> %s] Your package %s at %s expires at %s.%n",
            notification.customerId(), notification.packageId(),
            notification.locationAddress(), notification.expiryTime());
    }
}

/**
 * NotificationService manages observers and dispatches events.
 * Uses CopyOnWriteArrayList for thread-safe observer management.
 */
public class NotificationService {
    private final List<NotificationObserver> observers = new CopyOnWriteArrayList<>();

    public void subscribe(NotificationObserver observer) {
        observers.add(observer);
    }

    public void unsubscribe(NotificationObserver observer) {
        observers.remove(observer);
    }

    public void notifyPackageDeposited(PickupNotification notification) {
        for (NotificationObserver observer : observers) {
            // In production: dispatch asynchronously to avoid blocking
            observer.onPackageDeposited(notification);
        }
    }

    public void notifyExpiryReminder(PickupNotification notification) {
        for (NotificationObserver observer : observers) {
            observer.onExpiryReminder(notification);
        }
    }
}
Delivery - Linking Package, Locker, and OTP ▼ 
import java.time.Duration;
import java.time.LocalDateTime;
import java.util.UUID;

public class Delivery {
    private final String deliveryId;
    private final Package pkg;
    private final Locker locker;
    private final Customer customer;
    private final String locationId;
    private String otp;
    private LocalDateTime depositTime;
    private LocalDateTime pickupTime;
    private LocalDateTime expiryTime;
    private DeliveryStatus status;

    private static final Duration DEFAULT_EXPIRY = Duration.ofDays(3);

    public Delivery(Package pkg, Locker locker, Customer customer, String locationId) {
        this.deliveryId = UUID.randomUUID().toString().substring(0, 10).toUpperCase();
        this.pkg = pkg;
        this.locker = locker;
        this.customer = customer;
        this.locationId = locationId;
        this.status = DeliveryStatus.RESERVED;
    }

    public void markDeposited(String otp) {
        if (status != DeliveryStatus.RESERVED) {
            throw new IllegalStateException(
                "Cannot deposit: delivery " + deliveryId + " is " + status);
        }
        this.otp = otp;
        this.depositTime = LocalDateTime.now();
        this.expiryTime = depositTime.plus(DEFAULT_EXPIRY);
        this.status = DeliveryStatus.DEPOSITED;
        this.pkg.transitionTo(PackageStatus.DEPOSITED);
    }

    public void markPickedUp() {
        if (status != DeliveryStatus.DEPOSITED) {
            throw new IllegalStateException(
                "Cannot pick up: delivery " + deliveryId + " is " + status);
        }
        this.pickupTime = LocalDateTime.now();
        this.status = DeliveryStatus.PICKED_UP;
        this.pkg.transitionTo(PackageStatus.PICKED_UP);
    }

    public void markExpired() {
        if (status != DeliveryStatus.DEPOSITED) {
            throw new IllegalStateException(
                "Cannot expire: delivery " + deliveryId + " is " + status);
        }
        this.status = DeliveryStatus.EXPIRED;
        this.pkg.transitionTo(PackageStatus.EXPIRED);
    }

    public void markReturned() {
        if (status != DeliveryStatus.EXPIRED) {
            throw new IllegalStateException(
                "Cannot return: delivery " + deliveryId + " is " + status);
        }
        this.status = DeliveryStatus.RETURNED;
        this.pkg.transitionTo(PackageStatus.RETURNED);
    }

    public boolean isExpired() {
        return status == DeliveryStatus.DEPOSITED
            && expiryTime != null
            && LocalDateTime.now().isAfter(expiryTime);
    }

    // Getters
    public String getDeliveryId() { return deliveryId; }
    public Package getPackage() { return pkg; }
    public Locker getLocker() { return locker; }
    public Customer getCustomer() { return customer; }
    public String getLocationId() { return locationId; }
    public String getOtp() { return otp; }
    public LocalDateTime getDepositTime() { return depositTime; }
    public LocalDateTime getPickupTime() { return pickupTime; }
    public LocalDateTime getExpiryTime() { return expiryTime; }
    public DeliveryStatus getStatus() { return status; }

    public DeliveryInfo toInfo() {
        return new DeliveryInfo(deliveryId, pkg.getPackageId(),
            locker.getLockerId(), locationId, status, depositTime, expiryTime);
    }
}
LockerSystem - Main Controller ▼ 
import java.util.*;
import java.util.concurrent.ConcurrentHashMap;

/**
 * LockerSystem is the main orchestrator.
 *
 * Key responsibilities:
 * 1. Reserve a locker for a package (allocation strategy)
 * 2. Handle package deposit (OTP generation, notification)
 * 3. Handle customer pickup (OTP validation, locker release)
 * 4. Handle expired packages (scheduled scan, return-to-sender)
 *
 * Note: NOT a Singleton -- unlike parking lots, you may have
 * separate locker systems for different regions/tenants.
 */
public class LockerSystem {
    private final Map<String, LockerLocation> locations;
    private final Map<String, Delivery> deliveries; // deliveryId -> Delivery
    private final Map<String, Customer> customers;  // customerId -> Customer
    private final LockerAllocationStrategy allocationStrategy;
    private final OTPService otpService;
    private final NotificationService notificationService;

    public LockerSystem(LockerAllocationStrategy strategy) {
        this.locations = new ConcurrentHashMap<>();
        this.deliveries = new ConcurrentHashMap<>();
        this.customers = new ConcurrentHashMap<>();
        this.allocationStrategy = strategy;
        this.otpService = new OTPService();
        this.notificationService = new NotificationService();
    }

    public void addLocation(LockerLocation location) {
        locations.put(location.getLocationId(), location);
    }

    public void registerCustomer(Customer customer) {
        customers.put(customer.getCustomerId(), customer);
    }

    public NotificationService getNotificationService() {
        return notificationService;
    }

    // -------------------------------------------------------
    // 1. RESERVE a locker for an incoming package
    // -------------------------------------------------------

    /**
     * Reserves a locker at the given location for the package.
     * Uses the configured allocation strategy to pick the best locker.
     *
     * @throws IllegalArgumentException if location not found
     * @throws IllegalStateException if no locker available
     */
    public Delivery reserveLocker(String locationId, Package pkg, String customerId) {
        LockerLocation location = locations.get(locationId);
        if (location == null) {
            throw new IllegalArgumentException("Location not found: " + locationId);
        }

        Customer customer = customers.get(customerId);
        if (customer == null) {
            throw new IllegalArgumentException("Customer not found: " + customerId);
        }

        // Get available lockers and apply allocation strategy
        List<Locker> available = location.getAvailableLockers(pkg.getRequiredSize());
        Locker locker = allocationStrategy.allocate(available, pkg.getRequiredSize())
            .orElseThrow(() -> new IllegalStateException(
                "No locker available at " + locationId + " for size " + pkg.getRequiredSize()));

        // Reserve the locker (thread-safe)
        if (!locker.reserve()) {
            // Race condition: locker was taken between check and reserve
            throw new IllegalStateException("Locker reservation failed due to contention");
        }

        pkg.transitionTo(PackageStatus.IN_TRANSIT);

        Delivery delivery = new Delivery(pkg, locker, customer, locationId);
        deliveries.put(delivery.getDeliveryId(), delivery);

        return delivery;
    }

    // -------------------------------------------------------
    // 2. DEPOSIT a package into the reserved locker
    // -------------------------------------------------------

    /**
     * Called when a driver physically places the package in the locker.
     * Generates OTP and notifies the customer.
     */
    public boolean depositPackage(String deliveryId) {
        Delivery delivery = deliveries.get(deliveryId);
        if (delivery == null) {
            throw new IllegalArgumentException("Delivery not found: " + deliveryId);
        }

        Locker locker = delivery.getLocker();
        Package pkg = delivery.getPackage();

        // Place package in locker
        if (!locker.assignPackage(pkg)) {
            return false;
        }

        // Generate OTP
        String otp = otpService.generate(delivery.getLocationId());
        delivery.markDeposited(otp);

        // Register OTP at location for lookup during pickup
        LockerLocation location = locations.get(delivery.getLocationId());
        location.registerOTP(otp, delivery);

        // Notify customer
        PickupNotification notification = new PickupNotification(
            delivery.getCustomer().getCustomerId(),
            location.getAddress(),
            otp,
            delivery.getExpiryTime(),
            pkg.getPackageId()
        );
        notificationService.notifyPackageDeposited(notification);

        return true;
    }

    // -------------------------------------------------------
    // 3. PICKUP -- customer enters OTP at the kiosk
    // -------------------------------------------------------

    /**
     * Processes a customer pickup attempt.
     *
     * @param locationId the locker location
     * @param otp the code entered by the customer
     * @return the retrieved Package
     * @throws IllegalArgumentException if OTP is invalid
     */
    public Package pickupPackage(String locationId, String otp) {
        LockerLocation location = locations.get(locationId);
        if (location == null) {
            throw new IllegalArgumentException("Location not found: " + locationId);
        }

        // Validate OTP
        if (!otpService.validate(locationId, otp)) {
            throw new IllegalArgumentException("Invalid or expired OTP");
        }

        // Find the delivery associated with this OTP
        Delivery delivery = location.findDeliveryByOTP(otp);
        if (delivery == null) {
            throw new IllegalArgumentException("No delivery found for OTP");
        }

        // Release package from locker
        Locker locker = delivery.getLocker();
        Package pkg = locker.releasePackage();

        // Update delivery state
        delivery.markPickedUp();

        // Invalidate OTP
        otpService.invalidate(locationId, otp);
        location.invalidateOTP(otp);

        return pkg;
    }

    // -------------------------------------------------------
    // 4. EXPIRY HANDLING -- scheduled task
    // -------------------------------------------------------

    /**
     * Scans all active deliveries and marks expired ones.
     * In production, this runs as a scheduled task (e.g., every hour).
     *
     * Returns the list of expired deliveries for return-to-sender processing.
     */
    public List<Delivery> handleExpiredPackages() {
        List<Delivery> expiredDeliveries = new ArrayList<>();

        for (Delivery delivery : deliveries.values()) {
            if (delivery.isExpired()) {
                delivery.markExpired();

                // Invalidate OTP
                String otp = delivery.getOtp();
                otpService.invalidate(delivery.getLocationId(), otp);
                LockerLocation location = locations.get(delivery.getLocationId());
                if (location != null) {
                    location.invalidateOTP(otp);
                }

                expiredDeliveries.add(delivery);

                System.out.printf("EXPIRED: Delivery %s, Package %s at location %s%n",
                    delivery.getDeliveryId(),
                    delivery.getPackage().getPackageId(),
                    delivery.getLocationId());
            }
        }
        return expiredDeliveries;
    }

    /**
     * Called when a driver picks up an expired package for return.
     * Releases the locker back to the available pool.
     */
    public void processReturn(String deliveryId) {
        Delivery delivery = deliveries.get(deliveryId);
        if (delivery == null) {
            throw new IllegalArgumentException("Delivery not found: " + deliveryId);
        }

        delivery.markReturned();
        delivery.getLocker().releasePackage();

        System.out.printf("RETURNED: Package %s from locker %s%n",
            delivery.getPackage().getPackageId(),
            delivery.getLocker().getLockerId());
    }

    // -------------------------------------------------------
    // Query methods
    // -------------------------------------------------------

    public LocationAvailability getLocationAvailability(String locationId) {
        LockerLocation location = locations.get(locationId);
        if (location == null) {
            throw new IllegalArgumentException("Location not found: " + locationId);
        }
        return location.getAvailability();
    }

    public DeliveryInfo getDeliveryInfo(String deliveryId) {
        Delivery delivery = deliveries.get(deliveryId);
        if (delivery == null) {
            throw new IllegalArgumentException("Delivery not found: " + deliveryId);
        }
        return delivery.toInfo();
    }
}
Expiry Handling - Scheduled Task ▼

In production, package expiry is handled by a scheduled task that runs periodically. This keeps the core system clean -- expiry logic does not pollute the deposit/pickup paths.

import java.util.concurrent.Executors;
import java.util.concurrent.ScheduledExecutorService;
import java.util.concurrent.TimeUnit;

/**
 * Scheduled job that scans for expired packages.
 *
 * In production this might be:
 * - A cron job / Kubernetes CronJob
 * - AWS EventBridge + Lambda
 * - A Spring @Scheduled method
 *
 * Here we use ScheduledExecutorService for simplicity.
 */
public class ExpiryScheduler {
    private final LockerSystem lockerSystem;
    private final ScheduledExecutorService scheduler;

    public ExpiryScheduler(LockerSystem lockerSystem) {
        this.lockerSystem = lockerSystem;
        this.scheduler = Executors.newSingleThreadScheduledExecutor(r -> {
            Thread t = new Thread(r, "expiry-scanner");
            t.setDaemon(true);
            return t;
        });
    }

    /**
     * Starts the expiry scanner. Runs every hour by default.
     */
    public void start() {
        scheduler.scheduleAtFixedRate(() -> {
            try {
                var expired = lockerSystem.handleExpiredPackages();
                if (!expired.isEmpty()) {
                    System.out.printf("Expiry scan: %d packages expired%n",
                        expired.size());
                }
            } catch (Exception e) {
                System.err.println("Expiry scan error: " + e.getMessage());
            }
        }, 0, 1, TimeUnit.HOURS);
    }

    public void stop() {
        scheduler.shutdown();
    }
}

10. Scale-Out Considerations

Distributed Locker Network ▼

In a real Amazon-scale deployment, the single-process design above needs to evolve. Here is how each concern scales:

Geo-Based Allocation

  • Problem: When a customer places an order, the system must choose which locker location to assign (closest to the delivery address, with availability).
  • Solution: A separate LocationSelectionService queries a geo-index (e.g., PostGIS, Elasticsearch geo_point) for nearby locations, then checks real-time availability via a cache (Redis) before reserving.
  • Trade-off: Stale availability data might cause reservation failures; use optimistic locking with retry.

Distributed State

  • Locker state: Each physical kiosk maintains local state and syncs with a central service via event sourcing. If the network is down, the kiosk can still validate OTPs using a locally cached set.
  • Database: Deliveries table sharded by locationId. OTP lookups are always scoped to a single location, so no cross-shard queries.
  • Event-driven: State transitions publish events (Kafka/SQS) consumed by analytics, notification, and billing services.

Analytics and Dynamic Pricing

  • Usage analytics: Track utilization rates per location, peak hours, average pickup time. Feed into a dashboard for capacity planning.
  • Dynamic pricing: At high-demand locations, charge a premium for locker delivery. The AllocationStrategy can incorporate pricing signals.
  • Predictive pre-staging: ML model predicts which locations will fill up; proactively suggests alternative locations during checkout.

Reliability

  • Idempotent operations: Deposit and pickup are idempotent (re-calling with same deliveryId/OTP is safe).
  • Dead letter queue: Failed notifications go to a DLQ for retry.
  • Circuit breaker: If the notification service is down, package is still deposited; notification retries asynchronously.

11. Interview Tips

Common Follow-up Questions

  • "How would you handle multiple packages for the same customer at the same location?" -- Each package gets its own locker and OTP. The customer receives multiple notifications. The kiosk UI shows a list after validating any one OTP (stretch goal).
  • "What if the customer wants to return a package via the locker?" -- Add a RETURN_DEPOSIT state. Customer requests a return OTP, deposits package, driver picks it up. Reuse the existing state machine with a parallel return flow.
  • "How would you add a waitlist when all lockers are full?" -- Introduce a WaitlistQueue per location. When a locker becomes available, the queue is drained FIFO. Notify the next customer and hold the reservation for a time window.
  • "How do you prevent OTP brute-force attacks?" -- Rate limit at the kiosk (max 5 attempts per 15 minutes). Lock out after repeated failures. Consider adding CAPTCHA for the mobile app flow. Use 8-digit OTPs at high-traffic locations.
  • "How would you monitor this system?" -- Key metrics: locker utilization rate, average pickup time, expiry rate, OTP validation failure rate. Alert on anomalies (sudden spike in failures could indicate an attack or hardware issue).

Red Flags to Avoid

  • No state machine: Allowing arbitrary status changes (e.g., jumping from RESERVED to PICKED_UP) suggests lack of domain modeling rigor
  • Global OTPs: OTPs should be scoped per location, not globally unique -- explain why (reduces collision space, simplifies validation)
  • Ignoring concurrency: Two drivers depositing at the same location must not get the same locker. Use locks or CAS operations
  • No expiry handling: Forgetting the expiry flow means lockers get permanently stuck. Always design for the unhappy path
  • Hardcoded notification: Directly sending SMS in the deposit method violates Open/Closed -- use Observer or event bus
  • Singleton LockerSystem: Unlike a parking lot, locker systems span regions. Avoid unnecessary Singleton; explain the difference
