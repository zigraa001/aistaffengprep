URL Shortener

Design Overview ▼

Requirements

  • Shorten long URLs to ~7 characters
  • Redirect shortened URLs to original
  • Scale: 500M new URLs/month, 100:1 read/write ratio
flowchart LR C[Client] --> LB[Load Balancer] LB --> API[API Servers] API --> Cache[Redis Cache] API --> DB[(Database)] API --> Counter[ID Generator]

Key Decisions

  • ID Generation: Base62 encoding of counter or UUID
  • Database: NoSQL (simple key-value access)
  • Caching: Cache popular URLs (80/20 rule)

Estimation

Write QPS: 500M / (30 * 24 * 3600) ≈ 200/s
Read QPS: 200 * 100 = 20,000/s
Storage (5 years): 500M * 12 * 5 * 500B ≈ 15TB

Complete URL Shortener Service

# Python - URL Shortener with FastAPI from fastapi import FastAPI, HTTPException from fastapi.responses import RedirectResponse import redis import hashlib from typing import Optional app = FastAPI() redis_client = redis.Redis(host='localhost', port=6379, db=0) ALPHABET = "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz" def encode_base62(num: int) -> str: if num == 0: return ALPHABET[0] result = [] while num > 0: result.append(ALPHABET[num % 62]) num //= 62 return ''.join(reversed(result)).zfill(7) @app.post("/api/shorten") async def shorten_url(long_url: str, custom_alias: Optional[str] = None): # Custom alias if custom_alias: if redis_client.exists(f"url:{custom_alias}"): raise HTTPException(400, "Alias already exists") short_code = custom_alias else: # Generate using atomic counter counter = redis_client.incr("url_counter") short_code = encode_base62(counter) # Store mapping redis_client.set(f"url:{short_code}", long_url) return {"short_url": f"https://short.url/{short_code}"} @app.get("/{short_code}") async def redirect(short_code: str): long_url = redis_client.get(f"url:{short_code}") if not long_url: raise HTTPException(404, "URL not found") # Increment click counter (async in production) redis_client.incr(f"clicks:{short_code}") # 301 = permanent (browser caches, less analytics visibility) # 302 = temporary (every request hits server, better for click tracking) return RedirectResponse(url=long_url.decode(), status_code=302)

Distributed Rate Limiter with Redis

# Python - Sliding Window Counter Rate Limiter import time import redis class SlidingWindowRateLimiter: """ Sliding window counter - best balance of accuracy and memory. Smooths the boundary issue of fixed window. """ def __init__(self, redis_client: redis.Redis, limit: int, window_seconds: int): self.redis = redis_client self.limit = limit self.window = window_seconds def is_allowed(self, key: str) -> bool: now = time.time() current_window = int(now // self.window) prev_window = current_window - 1 position_in_window = (now % self.window) / self.window # Keys for current and previous window curr_key = f"rate:{key}:{current_window}" prev_key = f"rate:{key}:{prev_window}" # Use a Lua script to make check-and-increment atomic. # Without this, concurrent requests can race between the # GET (check) and INCR (increment), allowing burst over limit. lua_script = """ local prev_count = tonumber(redis.call('GET', KEYS[1]) or '0') local curr_count = tonumber(redis.call('GET', KEYS[2]) or '0') local weight = tonumber(ARGV[1]) local limit = tonumber(ARGV[2]) local ttl = tonumber(ARGV[3]) local weighted = prev_count * weight + curr_count if weighted >= limit then return 0 end redis.call('INCR', KEYS[2]) redis.call('EXPIRE', KEYS[2], ttl) return 1 """ result = self.redis.eval( lua_script, 2, # number of keys prev_key, curr_key, # KEYS[1], KEYS[2] 1 - position_in_window, # ARGV[1]: weight for prev window self.limit, # ARGV[2] self.window * 2 # ARGV[3]: TTL (keep for 2 windows) ) return bool(result) # Usage limiter = SlidingWindowRateLimiter(redis.Redis(), limit=100, window_seconds=60) if limiter.is_allowed(user_id): process_request() else: return_429_error()

Architecture ▼

Presence] end C1 <-->|WebSocket| WS1 C2 <-->|WebSocket| WS2 WS1 & WS2 --> MS MS --> MQ --> WS1 & WS2 MS --> DB WS1 & WS2 --> Cache

Key Components

  • WebSocket: Real-time bidirectional communication. Long-polling as fallback for environments that block WebSocket (corporate proxies, some mobile networks).
  • Message Queue: Decouple sending from delivery. Kafka or similar gives ordered, durable delivery per partition.
  • Presence Service: Track online/offline status. Heartbeat-based with a grace period (e.g., 30s) to avoid flapping on transient disconnects.
  • Message Store: Cassandra or similar for chat history, partitioned by (chat_id, message_timestamp) for efficient range queries.

Staff-Level: Failure Modes & Operational Concerns

  • Message ordering: A single chat’s messages must be ordered. Assign a monotonic sequence per conversation (e.g., Snowflake IDs) rather than relying on wall-clock time across servers.
  • Exactly-once delivery: Impossible in theory; in practice use idempotency keys on the client and server-side deduplication (message_id check before persisting).
  • WebSocket server failure: Clients reconnect to a different server. The new server must replay unacknowledged messages from the queue/store.
  • Group chat fan-out: For groups with thousands of members, writing N copies is expensive. Use a write-ahead approach: persist once, fan-out reads via message queue.
  • End-to-end encryption: Server cannot read message content. Key exchange (e.g., Signal Protocol / Double Ratchet) happens client-side; server stores opaque ciphertext.

Message Routing Implementation

# Python - Chat message routing with Redis pub/sub import asyncio import json import redis.asyncio as redis from dataclasses import dataclass from typing import Dict, Set @dataclass class ChatMessage: message_id: str sender_id: str recipient_id: str content: str timestamp: int class ChatRouter: def __init__(self, redis_url: str, server_id: str): self.redis = redis.from_url(redis_url) self.server_id = server_id self.local_connections: Dict[str, Set] = {} # user_id -> websockets async def register_connection(self, user_id: str, websocket): """Register a user's WebSocket connection on this server.""" if user_id not in self.local_connections: self.local_connections[user_id] = set() self.local_connections[user_id].add(websocket) # Store user -> server mapping in Redis await self.redis.hset("user:servers", user_id, self.server_id) # Subscribe to messages for this user pubsub = self.redis.pubsub() await pubsub.subscribe(f"user:{user_id}:messages") async def send_message(self, message: ChatMessage): """Route message to recipient.""" # 1. Persist to database (async) await self._persist_message(message) # 2. Find recipient's server recipient_server = await self.redis.hget("user:servers", message.recipient_id) # hget returns bytes in redis-py; decode for comparison if recipient_server: recipient_server = recipient_server.decode() if isinstance(recipient_server, bytes) else recipient_server if recipient_server == self.server_id: # Recipient on same server - deliver directly await self._deliver_locally(message) elif recipient_server: # Recipient on different server - use pub/sub await self.redis.publish( f"user:{message.recipient_id}:messages", json.dumps(message.__dict__) ) else: # Recipient offline - queue for later await self._queue_for_offline(message) async def _deliver_locally(self, message: ChatMessage): """Deliver to local WebSocket connections.""" connections = self.local_connections.get(message.recipient_id, set()) for ws in connections: await ws.send_json(message.__dict__) async def _queue_for_offline(self, message: ChatMessage): """Store message for offline delivery.""" await self.redis.lpush( f"user:{message.recipient_id}:pending", json.dumps(message.__dict__) )
